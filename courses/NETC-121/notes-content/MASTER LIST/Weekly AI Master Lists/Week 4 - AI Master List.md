## Canonical Concepts (deduplicated)
1. DHCP purpose: automatic client IP configuration (address, mask, gateway, DNS).
2. Dynamic vs static IP: pooled assignment vs manually fixed settings.
3. DHCP scope: per-subnet allocation boundaries and options.
4. DHCP reservation: fixed IP lease tied to device MAC.
5. DHCP lease timer: duration controls churn and address reuse behavior.
6. DORA flow: Discover, Offer, Request, Acknowledge.
7. DNS purpose: name-to-IP resolution at internet and enterprise scale.
8. FQDN model: host label + domain hierarchy identifies exact target.
9. DNS hierarchy: root, TLD, domain, host/subdomain levels.
10. DNS server roles: recursive resolution, authoritative responses, and caching.
11. DNS forwarder: upstream resolver for unresolved/non-local queries.
12. WINS context: legacy Microsoft name service preceding modern DNS usage.
13. Network latency: time delay metric measured in milliseconds.
14. Jitter: packet delay variation affecting real-time traffic quality.
15. Duplex model: simplex, half-duplex, full-duplex communication behavior.
16. RS-232 console fundamentals: start/stop/parity/data bits, baud rate, signal levels.
17. Multiplexing principle: multiple data streams share one medium.
18. FDM vs TDM: split by frequency bands vs split by time slots.
19. Binary addressing foundation: bit values define network/host boundaries in subnetting.
20. Wireshark workflow: installation, capture, and protocol visibility for packet analysis.
