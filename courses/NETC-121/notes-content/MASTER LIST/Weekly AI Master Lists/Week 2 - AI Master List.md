## Canonical Concepts (deduplicated)
1. OSI stack overview: Application, Presentation, Session, Transport, Network, Data Link, Physical.
2. Layer direction model: transmit path goes down stack; receive path goes up stack.
3. Layer-3 role: routers move packets between LANs using IP addressing.
4. Layer-2 role: switches/NICs deliver frames inside a LAN using MAC addresses.
5. Layer-1 role: physical signaling over media/connectors (for example RJ-45, copper, fiber).
6. IP address purpose: logical identifier used for host/network routing context.
7. IPv4 examples: private, loopback, and APIPA-style formats.
8. Classful addressing model: A/B/C ranges and default masks.
9. Reserved address behavior in subnet: all-zeros host bits = network address; all-ones host bits = broadcast.
10. MAC address structure: 48-bit address split into OUI (vendor) and NIC-specific portions.
11. ARP function: resolves IPv4 address to MAC address in local network.
12. RARP function: legacy reverse mapping from MAC to IP request model.
13. ARP table/cache: local database of IP-to-MAC mappings maintained by each device.
14. Protocol definition: communication rules for formatting, transmission, and processing.
15. Standard definition: formal published specification enabling interoperability.
16. Standards bodies: IEEE, IETF, ISO, and IANA roles in networking consistency.
