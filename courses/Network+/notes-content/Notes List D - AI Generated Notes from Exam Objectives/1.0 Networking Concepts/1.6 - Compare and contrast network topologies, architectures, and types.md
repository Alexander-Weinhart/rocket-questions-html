# 1.6 - Network Topologies, Architectures, and Types

> **Big idea:** Topology describes how a network is arranged. Architecture describes how the network is designed to operate. Network type describes the scope or purpose of the network, such as LAN, WAN, WLAN, or SAN.

## What you should be able to explain

- Compare star, mesh, spine-leaf, hub-and-spoke, and point-to-point designs.
- Understand LAN, WAN, WLAN, MAN, SAN, and SD-WAN at a high level.
- Explain client-server and peer-to-peer architectures.
- Recognize high availability and scalability tradeoffs.

## Concept map

| Design | What it looks like | Best use | Tradeoff |
| --- | --- | --- | --- |
| Star | Endpoints connect to a central switch | Common LAN access | Central device is critical |
| Mesh | Many interconnections | Resilience and multiple paths | Cost and complexity |
| Hub-and-spoke | Branches connect through a central hub | Simple WAN designs | Hub can bottleneck |
| Spine-leaf | Every leaf connects to every spine | Data centers | Requires consistent design discipline |
| Point-to-point | Direct link between two nodes | WAN or dedicated links | Limited to two endpoints |

## Network types

- A LAN connects devices in a limited area such as an office or building.
- A WLAN is a wireless LAN using access points and radio channels.
- A WAN connects sites over larger distances.
- A SAN is built for storage traffic and has different performance expectations than normal user access traffic.
- SD-WAN uses software policy to choose among WAN paths such as broadband, LTE/5G, and private circuits.

## Architecture choices

- Client-server centralizes services so clients request resources from servers.
- Peer-to-peer lets systems share directly, which can be simple but harder to manage at scale.
- Three-tier campus designs traditionally use access, distribution, and core layers.
- Collapsed core designs combine layers when the environment is smaller.

## Availability patterns

- Redundant links reduce single points of failure but can create loops without control mechanisms.
- Load sharing can improve performance and resilience.
- Overbuilding every part of a network is expensive, so match redundancy to business impact.

## Scenario walkthrough

1. A data center needs predictable east-west traffic between many server racks.
2. A spine-leaf topology is a strong fit because each leaf has consistent paths through the spine layer.
3. This reduces bottlenecks compared with designs that force too much traffic through one central point.

## Exam cues

- Topology questions often hide the answer in the traffic pattern.
- LAN/WAN/WLAN terms describe scope and medium more than brand or device type.
- High availability usually trades cost and complexity for resilience.
