# 1.8 - Modern Network Environments

> **Big idea:** Modern networks are more programmable, distributed, and security-aware than older static designs. Software-defined networking, infrastructure as code, zero trust, IPv6, overlays, and automation all exist to make networks more scalable, repeatable, and adaptable.

## What you should be able to explain

- Explain SDN, SD-WAN, VXLAN, zero trust, SASE, and IaC at a practical level.
- Understand why overlays and automation are used.
- Recognize how IPv6 changes addressing and design assumptions.
- Connect modern concepts to real operational problems.

## Concept map

| Concept | Plain meaning | Why it matters |
| --- | --- | --- |
| SDN | Control is abstracted from individual devices | Central policy and programmability |
| SD-WAN | Software-managed WAN path selection | Better use of multiple WAN circuits |
| VXLAN | Layer 2 overlay over Layer 3 networks | Scalable segmentation in data centers |
| Zero trust | Verify explicitly instead of trusting location | Limits implicit network trust |
| SASE | Security and access delivered as cloud edge services | Supports distributed users and apps |
| IaC | Infrastructure described in versioned code | Repeatable builds and change review |

## Why overlays exist

- Traditional VLANs have scale and location limits.
- Overlays let logical networks span across an IP fabric.
- Encapsulation adds headers, so MTU planning matters.
- VXLAN is common in data centers because it supports large-scale segmentation.

## Automation and infrastructure as code

- Manual device-by-device changes are slow and error-prone.
- Templates and code make configuration repeatable.
- Version control gives teams review history and rollback context.
- Bad automation can break things quickly, so testing and staged rollout matter.

## Zero trust thinking

- A user or device should not be trusted only because it is on an internal network.
- Access decisions should consider identity, device posture, policy, and context.
- Segmentation limits what an attacker can reach if one account or endpoint is compromised.

## Scenario walkthrough

1. A company has users everywhere and applications split between SaaS, data centers, and cloud networks.
2. A perimeter-only model becomes awkward because traffic no longer always returns to one headquarters firewall.
3. SASE and zero trust concepts help move inspection and policy closer to the user and application.

## Exam cues

- Modern does not always mean better for every use case. Match the tool to the problem.
- VXLAN is an overlay; SD-WAN is about WAN path control and policy.
- IaC is about repeatability, review, and consistency.
