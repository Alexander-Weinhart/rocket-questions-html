# 1.7 - IPv4 Network Addressing

> **Big idea:** IPv4 addressing gives hosts a logical identity and tells them which destinations are local versus remote. The address, subnet mask, default gateway, and DNS settings must work together or communication fails in ways that look mysterious until you break them down.

## What you should be able to explain

- Explain IPv4 addresses, masks, networks, hosts, and gateways.
- Understand private addressing, APIPA, loopback, and special-use ranges.
- Use subnetting ideas to determine local and remote destinations.
- Recognize symptoms of common IPv4 misconfiguration.

## Concept map

| Concept | Meaning | Example clue |
| --- | --- | --- |
| IP address | Host logical address | Assigned manually or by DHCP |
| Subnet mask | Divides network bits from host bits | Wrong mask causes local/remote confusion |
| Default gateway | Router used for nonlocal traffic | Can reach local hosts but not other networks |
| DNS server | Resolves names to addresses | Can ping IP but not name |
| DHCP | Automatically supplies IP settings | Lease, scope, reservation, helper |

## Local versus remote

- A host compares the destination IP to its own IP and subnet mask.
- If the destination is local, the host uses ARP to find the destination MAC address.
- If the destination is remote, the host sends the frame to the default gateway MAC address.
- The IP packet still has the final destination IP; the Layer 2 frame changes at each hop.

## Special address ranges

- Private IPv4 ranges are used internally and are commonly translated by NAT.
- Loopback addresses test the local TCP/IP stack.
- APIPA addresses appear when a host cannot obtain DHCP but auto-assigns a link-local address.
- Network and broadcast addresses cannot be assigned to normal hosts in traditional IPv4 subnets.

## Subnetting without panic

- CIDR notation tells you how many bits are network bits.
- More network bits means smaller subnets and fewer hosts per subnet.
- Fewer network bits means larger subnets and more hosts per subnet.
- The exam often wants the correct troubleshooting interpretation more than long arithmetic.

## Scenario walkthrough

1. Two hosts are plugged into the same switch. One uses a mask that makes the other host look remote.
2. The confused host sends traffic to the default gateway instead of ARPing directly for the local host.
3. If the gateway does not route that traffic back correctly, communication fails even though both hosts are physically nearby.

## Exam cues

- Wrong mask is a classic hidden cause.
- If a host has an APIPA address, investigate DHCP reachability.
- If IP works but names fail, suspect DNS before replacing cables.
