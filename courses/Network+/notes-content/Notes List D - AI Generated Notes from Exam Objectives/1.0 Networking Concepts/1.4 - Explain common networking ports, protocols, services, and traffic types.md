# 1.4 - Ports, Protocols, Services, and Traffic Types

> **Big idea:** Ports and protocols tell you what kind of conversation is happening. A port identifies a service endpoint at Layer 4, while a protocol defines the rules of the exchange. Traffic types describe how many recipients are intended and how the network should deliver the packets.

## What you should be able to explain

- Recognize common service ports and protocol names.
- Compare TCP, UDP, ICMP, GRE, and IPSec components.
- Explain unicast, multicast, broadcast, and anycast.
- Use port knowledge for firewall and troubleshooting decisions.

## Concept map

| Service | Port or ports | Transport | What it is for |
| --- | --- | --- | --- |
| FTP | 20/21 | TCP | Transfers files between systems. Port 21 handles commands and login, while port 20 is traditionally used for active-mode data transfer. It is legacy because traffic is not encrypted by default. |
| SSH | 22 | TCP | Provides encrypted remote command-line administration. Administrators use it to securely log in to network devices, Linux servers, and other managed systems. |
| SFTP | 22 | TCP | Transfers files through an encrypted SSH session. It is commonly used when file transfer needs authentication and confidentiality. |
| Telnet | 23 | TCP | Provides remote terminal access without encryption. It is mostly seen in legacy equipment, labs, or exam questions as the insecure predecessor to SSH. |
| SMTP | 25 | TCP | Moves email between mail servers. Clients may also submit outbound mail through SMTP submission ports, but port 25 is the classic server-to-server mail transfer port. |
| DNS | 53 | UDP/TCP | Translates names into IP addresses and supports other name-service records. UDP is common for normal queries, while TCP is used for zone transfers and some large responses. |
| DHCP | 67/68 | UDP | Automatically gives hosts IP configuration such as address, mask, gateway, and DNS settings. Servers listen on UDP 67, and clients use UDP 68. |
| TFTP | 69 | UDP | Transfers files with a very simple, unauthenticated protocol. It is often used for network device boot files, firmware images, or configuration recovery in controlled networks. |
| HTTP | 80 | TCP | Carries unencrypted web traffic. Browsers and web clients use it to request pages, APIs, and other web resources. |
| NTP | 123 | UDP | Synchronizes clocks between systems. Accurate time is important for logs, certificates, authentication, and event correlation. |
| SNMP | 161/162 | UDP | Monitors and manages network devices. Managers query agents on UDP 161, and devices send traps or alerts to UDP 162. |
| LDAP | 389 | TCP/UDP | Queries and updates directory services, such as users, groups, and authentication-related objects. TCP is the common transport for normal directory operations. |
| HTTPS | 443 | TCP | Carries encrypted web traffic using TLS. It protects web sessions, APIs, credentials, and data in transit. |
| SMB | 445 | TCP | Provides file sharing, printer sharing, and Windows network resource access. It is common in internal networks and tightly controlled across firewalls. |
| Syslog | 514 | UDP | Sends log messages from devices and systems to a central log collector. UDP 514 is the traditional syslog port, though secured deployments may use other transports. |
| SMTPS / SMTP submission | 587 | TCP | Submits outbound email from authenticated mail clients to a mail server, usually with STARTTLS encryption. It is different from SMTP port 25, which is mainly server-to-server. |
| LDAPS | 636 | TCP | Provides LDAP directory access over TLS. It protects directory queries and authentication-related traffic from being read in transit. |
| SQL Server | 1433 | TCP | Provides database access to Microsoft SQL Server. Applications use it to connect to SQL databases, so firewall rules should limit it to trusted clients. |
| RDP | 3389 | TCP/UDP | Provides graphical remote desktop access to Windows systems. TCP handles the main session, and UDP can improve performance for modern RDP sessions. |
| SIP | 5060/5061 | TCP/UDP | Sets up, manages, and tears down voice or video sessions. Port 5060 is commonly unencrypted SIP, while 5061 is commonly SIP over TLS. |

## TCP versus UDP

- TCP is connection-oriented and tracks sequencing, acknowledgments, retransmission, and flow control.
- UDP is connectionless and has lower overhead, which is useful for voice, video, DNS queries, and other traffic where the application can handle loss or retry.
- A firewall rule must account for the right transport, not only the port number.

## IP-related protocols

- ICMP carries diagnostics and control messages such as echo requests and unreachable messages.
- GRE encapsulates traffic inside another IP packet and is often seen with tunneling designs.
- IPSec protects IP traffic using AH, ESP, and IKE-related negotiation.
- ESP can provide confidentiality, integrity, and authentication; AH focuses on integrity and authentication.

## Traffic delivery types

- Unicast is one sender to one receiver and is the most common traffic pattern.
- Broadcast is one sender to every host in the local broadcast domain.
- Multicast is one sender to a subscribed group.
- Anycast uses the same destination address in multiple places, with routing delivering traffic to the nearest or best instance.

## Scenario walkthrough

1. A web server works locally but fails from another subnet after a firewall change.
2. Confirm the expected service first: HTTPS normally means TCP port 443.
3. Then test whether routing works and whether TCP 443 is allowed through the firewall path.
4. Do not open every port. Fix the exact service path.

## Exam cues

- Memorize the common ports, but also know the service purpose.
- DNS can use UDP and TCP. That nuance appears in troubleshooting questions.
- Broadcast stays in the local broadcast domain unless a relay/helper feature is involved.
