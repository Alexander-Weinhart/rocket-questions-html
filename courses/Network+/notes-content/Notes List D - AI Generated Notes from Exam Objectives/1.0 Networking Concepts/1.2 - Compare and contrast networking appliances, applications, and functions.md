# 1.2 - Networking Appliances, Applications, and Functions

> **Big idea:** Network appliances and applications are the building blocks that move, secure, inspect, and optimize traffic. The exam expects you to recognize what each device or function does, where it usually sits, and what problem it is designed to solve.

## What you should be able to explain

- Compare routers, switches, firewalls, load balancers, proxies, and wireless infrastructure.
- Understand where security and inspection appliances fit.
- Recognize network services such as DHCP, DNS, NTP, and IPAM.
- Separate device names from the functions they provide.

## Concept map

| Component | Primary role | Best clue in a scenario |
| --- | --- | --- |
| Router | Moves traffic between IP networks | Default gateways, WAN links, routing tables |
| Switch | Forwards Ethernet frames inside a LAN | VLANs, trunks, MAC address table |
| Firewall | Allows or blocks traffic by policy | ACLs, zones, stateful inspection, rules |
| Load balancer | Distributes sessions across back-end systems | Server pools, health checks, virtual IPs |
| Proxy | Makes requests on behalf of clients | Filtering, caching, inspection, forward/reverse proxy |
| IDS/IPS | Detects or blocks suspicious traffic | Signatures, alerts, inline prevention |
| Wireless controller | Centralizes AP configuration | SSID deployment, roaming, channel plans |

## Appliance versus function

- A physical or virtual box may perform several functions at once.
- A firewall can also route, perform NAT, terminate VPNs, and inspect applications.
- A Layer 3 switch can switch inside VLANs and route between VLANs.
- In exam scenarios, identify the job being described instead of relying only on the product name.

## Common network services

- DHCP gives clients IP settings such as address, mask, gateway, and DNS servers.
- DNS maps names to records such as A, AAAA, CNAME, MX, TXT, and SRV.
- NTP keeps system clocks synchronized, which matters for logs, certificates, and authentication.
- IPAM tracks address allocations so teams do not rely on spreadsheets and guesswork.

## Security and visibility functions

- IDS watches and alerts; IPS sits inline and can block.
- A content filter limits access by category, reputation, or policy.
- A VPN concentrator terminates encrypted tunnels for remote users or sites.
- Packet brokers, taps, and port mirrors help monitoring tools see traffic.

## Scenario walkthrough

1. A company wants one public address to represent several internal web servers and automatically stop sending users to a failed server.
2. That is a load-balancing scenario.
3. The key clues are a front-end virtual address, a pool of real servers, and health checks.
4. A firewall rule may still be required, but the traffic-distribution function is the load balancer.

## Exam cues

- Watch for verbs: route, switch, inspect, balance, resolve, assign, synchronize.
- Many appliances are virtual now. The function matters more than whether it is hardware or software.
- IDS is detection; IPS is prevention. That tiny letter changes the answer.
