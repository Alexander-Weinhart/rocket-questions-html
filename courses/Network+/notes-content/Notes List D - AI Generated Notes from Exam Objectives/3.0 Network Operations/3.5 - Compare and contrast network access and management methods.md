# 3.5 - Network Access and Management Methods

> **Big idea:** Management access is how administrators configure and support devices. Good management design uses secure protocols, controlled access paths, authentication, authorization, accounting, and separation from ordinary user traffic.

## What you should be able to explain

- Compare in-band and out-of-band management.
- Understand SSH, HTTPS, console, API, SNMP, and remote access methods.
- Explain AAA, RADIUS, TACACS+, and least privilege.
- Recognize secure management design choices.

## Concept map

| Method | Use | Security note |
| --- | --- | --- |
| Console | Direct local device access | Strong for recovery but requires physical/control access |
| SSH | Encrypted CLI management | Preferred over Telnet |
| HTTPS GUI | Encrypted web management | Use certificates and strong auth |
| SNMPv3 | Secure monitoring/management | Better than older community-string models |
| API | Automation and integration | Protect tokens and permissions |
| Out-of-band | Separate management path | Useful during production network failures |

## In-band versus out-of-band

- In-band management uses the production network path.
- Out-of-band management uses a separate path such as console servers, management networks, or cellular backup.
- Out-of-band access is valuable when the production network is broken.
- In-band access is cheaper and common but depends on the network being healthy.

## AAA

- Authentication verifies identity.
- Authorization decides what the authenticated identity is allowed to do.
- Accounting records what happened.
- RADIUS is common for network access; TACACS+ is common for detailed device administration control.

## Management hardening

- Disable insecure services such as Telnet when SSH is available.
- Restrict management access by source network and role.
- Use centralized identity where appropriate.
- Log administrative actions and keep time synchronized.

## Scenario walkthrough

1. A remote router is reachable by users, but administrators cannot manage it after an ACL change.
2. That is an in-band management risk.
3. An out-of-band path or console server could allow recovery without waiting for someone on site.

## Exam cues

- Telnet is the insecure legacy answer; SSH is the secure CLI answer.
- AAA is authentication, authorization, and accounting.
- Out-of-band management is about separate recovery/control access.
