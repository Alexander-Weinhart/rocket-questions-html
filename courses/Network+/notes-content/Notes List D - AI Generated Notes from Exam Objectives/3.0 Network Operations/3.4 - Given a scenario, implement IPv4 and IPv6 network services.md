# 3.4 - IPv4 and IPv6 Network Services

> **Big idea:** Network services provide the supporting functions that make addressing, naming, and time work. DHCP supplies addresses, DNS supplies names, NTP supplies trustworthy time, and IPv6 adds new addressing and autoconfiguration behaviors.

## What you should be able to explain

- Explain DHCP scopes, reservations, options, and relay/helpers.
- Understand DNS record types and resolution behavior.
- Explain IPv6 addressing, SLAAC, DHCPv6, and neighbor discovery.
- Recognize time-service importance for logs and authentication.

## Concept map

| Service | Purpose | Important details |
| --- | --- | --- |
| DHCPv4 | Assigns IPv4 settings | Scope, lease, reservation, options, relay |
| DNS | Resolves names and service records | A, AAAA, CNAME, MX, TXT, PTR, SRV |
| NTP | Synchronizes clocks | Logs, certificates, Kerberos-like auth, troubleshooting timelines |
| SLAAC | IPv6 host self-addressing | Uses router advertisements |
| DHCPv6 | IPv6 managed addressing/options | Stateful or stateless depending on design |

## DHCP details

- A scope defines the pool of addresses available for leases.
- A reservation maps a client identifier or MAC to a predictable address.
- Options provide extra settings such as gateway, DNS servers, and domain name.
- A DHCP relay/helper forwards client broadcasts to a DHCP server on another network.

## DNS details

- A and AAAA records map names to IPv4 and IPv6 addresses.
- CNAME records alias one name to another name.
- MX records identify mail exchangers.
- PTR records support reverse lookups.
- TXT records store text values used for verification and mail security.

## IPv6 basics

- IPv6 uses 128-bit addresses written in hexadecimal.
- Neighbor Discovery replaces many ARP-like functions.
- Link-local addresses are required for local IPv6 communication.
- Router advertisements help hosts discover prefixes and gateways.

## Scenario walkthrough

1. Clients on one VLAN get DHCP addresses, but clients on a new VLAN do not.
2. If the DHCP server is on a different subnet, check the relay/helper configuration on the gateway for the new VLAN.
3. The DHCP server may be healthy while the forwarding path is incomplete.

## Exam cues

- Can ping IP but not name points toward DNS.
- Wrong time can break authentication and make logs misleading.
- IPv6 does not use ARP; it uses Neighbor Discovery.
