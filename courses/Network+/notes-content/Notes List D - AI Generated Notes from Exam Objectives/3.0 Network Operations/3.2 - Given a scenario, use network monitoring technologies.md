# 3.2 - Network Monitoring Technologies

> **Big idea:** Monitoring turns network behavior into evidence. Instead of waiting for users to report pain, monitoring tools collect metrics, logs, flows, packets, and alerts so teams can understand health and respond faster.

## What you should be able to explain

- Compare SNMP, syslog, flow data, packet capture, and alerting.
- Understand metrics such as bandwidth, latency, jitter, errors, and discards.
- Use baselines and thresholds wisely.
- Recognize monitoring placement and visibility limits.

## Concept map

| Technology | What it shows | Best use |
| --- | --- | --- |
| SNMP | Device counters and status | Interface utilization, errors, device health |
| Syslog | Event messages from systems | Config changes, authentication, failures |
| NetFlow/sFlow/IPFIX | Traffic conversations and volumes | Who talked to whom, how much, and when |
| Packet capture | Actual packet details | Deep protocol troubleshooting |
| Synthetic monitoring | Simulated user or service checks | Verify application path availability |

## Metrics that matter

- Bandwidth utilization shows how much capacity is being used.
- Latency measures delay.
- Jitter measures variation in delay and matters for voice/video.
- Packet loss indicates dropped traffic and can crush TCP performance.
- Interface errors and discards often point to physical, duplex, congestion, or buffer issues.

## Logs versus metrics

- Metrics answer "how much" and "how often."
- Logs answer "what happened" and often include event context.
- Flow data summarizes conversations without full packet payloads.
- Packet captures provide detail but require careful filtering and interpretation.

## Alert quality

- Thresholds should reflect normal baselines and business impact.
- Too many noisy alerts teach people to ignore the system.
- Good alerts include device, symptom, severity, time, and likely next action.

## Scenario walkthrough

1. Users complain that voice calls sound choppy, but bandwidth graphs are not full.
2. Check jitter, packet loss, QoS markings, and interface errors.
3. Voice can fail from delay variation even when average utilization looks acceptable.

## Exam cues

- SNMP counters are great for interface health; packet capture is for detailed proof.
- NetFlow-like tools show conversations, not full payloads.
- Monitoring is only useful where it has visibility. Placement matters.
