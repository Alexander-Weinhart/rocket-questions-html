# 5.3 - Network Services Issues

> **Big idea:** Network services such as DHCP, DNS, routing, NAT, NTP, and authentication glue the network together. When they fail, users may say "the network is broken" even though cables and switches are fine.

## What you should be able to explain

- Troubleshoot DHCP, DNS, gateway, NAT, ACL, and time issues.
- Read symptoms to identify the failed service.
- Separate reachability from name resolution and authentication.
- Use targeted tests instead of broad guessing.

## Concept map

| Symptom | Likely service area | Useful test |
| --- | --- | --- |
| APIPA address | DHCP failure | Check scope, relay/helper, server reachability |
| Can ping IP not name | DNS failure | Query DNS, check records/resolver |
| Can reach local not remote | Gateway/routing | Check default gateway and routes |
| Some ports fail | Firewall/ACL/service listener | Test specific port and policy |
| Auth errors after clock drift | Time service | Check NTP and clock skew |

## DHCP troubleshooting

- Confirm whether the client has a lease from the expected scope.
- Check scope exhaustion, reservations, exclusions, and options.
- If clients are on another subnet, verify relay/helper configuration.
- Rogue DHCP can supply wrong gateways or DNS servers.

## DNS troubleshooting

- Check whether the resolver can answer the expected record type.
- Confirm the client is using the intended DNS server.
- Flush/cache behavior can hide recent changes.
- Split DNS can return different answers depending on client location.

## Policy and path issues

- Routing determines where packets go.
- ACLs and firewalls decide whether packets are allowed.
- NAT may change addresses in a way that affects return traffic or logging.
- A service can be reachable at Layer 3 but blocked at Layer 4.

## Scenario walkthrough

1. A laptop receives an address, but the gateway option is blank.
2. That is a DHCP option problem, not a switch cable problem.
3. Fix the DHCP scope options or policy so clients learn the correct default gateway.

## Exam cues

- IP works but names fail: DNS.
- No valid address: DHCP path or scope.
- Time problems can break authentication and certificates.
