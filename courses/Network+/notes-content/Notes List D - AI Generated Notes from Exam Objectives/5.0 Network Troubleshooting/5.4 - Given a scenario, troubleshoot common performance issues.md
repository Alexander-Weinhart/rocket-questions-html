# 5.4 - Network Performance Issues

> **Big idea:** Performance troubleshooting asks why something is slow, unstable, or inconsistent. The cause might be congestion, packet loss, latency, jitter, hardware limits, wireless contention, bad cabling, poor routing, DNS delay, or an overloaded application.

## What you should be able to explain

- Explain bandwidth, throughput, latency, jitter, and packet loss.
- Recognize congestion, errors, saturation, and oversubscription.
- Troubleshoot wireless performance and roaming issues.
- Use baselines and monitoring to prove performance problems.

## Concept map

| Metric | Meaning | Why users care |
| --- | --- | --- |
| Bandwidth | Theoretical or provisioned capacity | Upper limit of a link |
| Throughput | Actual delivered data rate | Real transfer speed |
| Latency | Delay from source to destination | Responsiveness |
| Jitter | Variation in delay | Voice/video smoothness |
| Packet loss | Dropped traffic | Retransmissions and choppy media |
| Errors/discards | Interface-level problems | Corruption, congestion, or hardware issues |

## Congestion and oversubscription

- Links can be designed with more downstream demand than upstream capacity.
- That is normal until peak usage exceeds what the uplink can handle.
- QoS can prioritize important traffic but does not create unlimited bandwidth.
- Monitoring helps prove whether utilization aligns with complaints.

## Latency and jitter

- Latency affects interactive applications such as voice, video, remote desktop, and gaming.
- Jitter hurts real-time media because packets arrive unevenly.
- WAN path changes, queueing, and wireless contention can all increase jitter.
- Average ping alone may hide jitter and loss.

## Wireless performance

- Signal strength is only one part of performance.
- Channel utilization, co-channel interference, client capabilities, and roaming behavior matter.
- Too-wide channels can reduce reliability in crowded areas.
- A Wi-Fi analyzer can reveal RF problems that IP tools cannot.

## Scenario walkthrough

1. File downloads are fast after hours but slow every afternoon.
2. That pattern suggests congestion or oversubscription during peak use.
3. Check interface utilization, errors, top talkers, and whether QoS or capacity planning is needed.

## Exam cues

- Throughput is what you actually get; bandwidth is what the link claims or is provisioned for.
- Jitter matters especially for voice and video.
- Performance answers should be evidence-based, not "upgrade everything" by default.
