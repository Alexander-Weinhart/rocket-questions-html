# 4.3 - Security Features, Defense Techniques, and Solutions

> **Big idea:** Security solutions are selected based on the asset, threat, and required business function. The best answer usually preserves needed access while reducing exposure through segmentation, inspection, authentication, encryption, monitoring, or hardening.

## What you should be able to explain

- Apply ACLs, firewall rules, VPNs, IDS/IPS, NAC, and segmentation.
- Understand switch security features such as port security, DHCP snooping, and dynamic ARP inspection.
- Recognize secure design patterns such as DMZs and screened subnets.
- Match controls to scenario clues.

## Concept map

| Control | Best fit | What it reduces |
| --- | --- | --- |
| Firewall policy | Filtering between zones/networks | Unauthorized network access |
| ACL | Specific traffic filtering on devices | Overbroad reachability |
| VPN | Protected traffic over untrusted networks | Exposure of data in transit |
| IDS/IPS | Detecting or blocking suspicious traffic | Known attack patterns |
| NAC | Controlling endpoint network admission | Untrusted/unhealthy devices |
| DMZ | Exposing public services safely | Direct access to internal networks |
| Port security | Limiting switch-port MAC behavior | Unauthorized device attachment |

## Defense on switches

- Port security can limit how many MAC addresses appear on a port.
- DHCP snooping marks trusted ports and blocks unauthorized DHCP replies.
- Dynamic ARP inspection uses trusted information to reduce ARP spoofing.
- BPDU guard protects edge ports from unexpected spanning-tree participation.

## Perimeter and internal controls

- A DMZ hosts services that must be reachable from less trusted networks.
- Internal segmentation firewalls control east-west traffic.
- ACLs can be useful near the source or destination when simple filtering is enough.
- IDS/IPS placement determines what traffic it can see or block.

## Secure access

- VPNs protect remote or site-to-site traffic.
- MFA strengthens identity checks.
- Certificate-based authentication can improve trust decisions.
- Least privilege should shape both user access and network reachability.

## Scenario walkthrough

1. A public web server needs internet access, but the database behind it must not be reachable from the internet.
2. Put the web server in a DMZ or public-facing segment.
3. Allow only the required application path from web to database.
4. Deny direct internet-to-database access.

## Exam cues

- Pick the narrowest control that satisfies the scenario.
- DMZ is for exposed services, not for random internal clients.
- DHCP snooping helps stop rogue DHCP; DAI helps stop ARP poisoning.
