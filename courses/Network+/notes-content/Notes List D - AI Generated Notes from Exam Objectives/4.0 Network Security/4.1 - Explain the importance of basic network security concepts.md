# 4.1 - Basic Network Security Concepts

> **Big idea:** Network security protects confidentiality, integrity, availability, and trust. It uses layered controls such as segmentation, access control, authentication, encryption, monitoring, hardening, and policy enforcement.

## What you should be able to explain

- Explain CIA, least privilege, defense in depth, and zero trust.
- Understand segmentation, ACLs, firewalls, and secure protocols.
- Recognize authentication, authorization, and accounting concepts.
- Connect risk, threats, vulnerabilities, and controls.

## Concept map

| Concept | Plain meaning | Network example |
| --- | --- | --- |
| Confidentiality | Keep data from unauthorized viewing | Encryption, access control |
| Integrity | Prevent or detect unauthorized change | Hashes, signatures, secure protocols |
| Availability | Keep services usable | Redundancy, DDoS protection, backups |
| Least privilege | Only grant needed access | Role-based firewall/admin permissions |
| Defense in depth | Multiple layers of control | Endpoint, network, identity, monitoring |
| Segmentation | Limit movement and blast radius | VLANs, ACLs, firewall zones |

## Access control

- ACLs and firewall policies define allowed traffic.
- Implicit deny means traffic is blocked unless explicitly permitted.
- Rules should be specific enough to allow business traffic without opening unnecessary paths.
- Network access control can evaluate device identity and posture before allowing access.

## Secure protocols

- Use SSH instead of Telnet.
- Use HTTPS instead of HTTP for management portals.
- Use SNMPv3 instead of older weak community-string approaches where possible.
- Use VPNs or private access methods when traffic crosses untrusted networks.

## Segmentation and trust boundaries

- Flat networks let problems spread easily.
- Segmentation divides systems by role, sensitivity, or function.
- Firewall zones make policy easier to reason about.
- Zero trust reduces assumptions based only on network location.

## Scenario walkthrough

1. A guest wireless network should provide internet access but no internal server access.
2. Use segmentation plus firewall rules or ACLs between the guest VLAN and internal networks.
3. The goal is availability for guests without extending internal trust.

## Exam cues

- Security questions often ask for the control that best reduces risk without breaking the required function.
- Least privilege applies to users, services, firewall rules, and administrators.
- Segmentation is one of the most common ways to limit lateral movement.
