# 2.1 - Routing Technologies

> **Big idea:** Routing is the process of moving packets between IP networks. A router chooses a next hop by consulting its routing table, which can be built from connected networks, static routes, dynamic routing protocols, and default routes.

## What you should be able to explain

- Explain route selection and next-hop behavior.
- Compare static, dynamic, and default routes.
- Understand administrative distance, metrics, and route summarization.
- Recognize NAT, PAT, and first-hop redundancy concepts.

## Concept map

| Routing idea | Meaning | Typical clue |
| --- | --- | --- |
| Connected route | Network directly attached to an interface | Appears when interface is up with an IP |
| Static route | Manually configured path | Simple, controlled, but not adaptive by itself |
| Default route | Used when no better match exists | Gateway of last resort |
| Dynamic route | Learned from routing protocol | Adapts to topology changes |
| Metric | Protocol-specific path cost | Lower cost is usually preferred |
| Administrative distance | Trust ranking between route sources | Used when sources disagree |

## Longest prefix match

- Routers choose the most specific matching route, not simply the first route listed.
- A route for a smaller subnet beats a broader summary route when both match.
- The default route is the least specific route.

## Static versus dynamic routing

- Static routing is predictable and simple for small or controlled paths.
- Dynamic routing learns routes and reacts to failures, which helps larger networks.
- Dynamic routing adds protocol overhead and requires design discipline.
- Route summarization reduces routing table size and hides unnecessary detail.

## NAT and PAT

- NAT changes IP addresses as traffic crosses a boundary.
- PAT allows many internal hosts to share one outside address by tracking transport ports.
- NAT is common at internet edges, but it can complicate troubleshooting because addresses change in transit.

## Scenario walkthrough

1. A branch has one internet circuit and no other paths.
2. A default static route to the ISP is probably enough.
3. A larger enterprise with multiple WAN paths benefits from dynamic routing or SD-WAN policy because failures and path selection matter.

## Exam cues

- Longest prefix match beats administrative distance after routes are candidates for the same destination.
- A default route is not magic; it only catches traffic with no better match.
- PAT is many-to-one translation using port tracking.
