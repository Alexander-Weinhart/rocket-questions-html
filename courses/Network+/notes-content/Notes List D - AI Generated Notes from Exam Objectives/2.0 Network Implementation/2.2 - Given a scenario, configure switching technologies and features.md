# 2.2 - Switching Technologies and Features

> **Big idea:** Switching controls local Ethernet delivery. Switches learn MAC addresses, forward frames, separate networks with VLANs, carry multiple VLANs over trunks, and use loop-prevention features so redundant links do not melt the Layer 2 domain.

## What you should be able to explain

- Explain MAC learning and forwarding.
- Configure and reason about access ports, trunk ports, and VLANs.
- Understand STP, link aggregation, and port security at a practical level.
- Recognize common switching misconfigurations.

## Concept map

| Feature | Purpose | Common symptom if wrong |
| --- | --- | --- |
| Access port | Carries one VLAN for an endpoint | Host lands in wrong network |
| Trunk port | Carries multiple VLANs between devices | VLAN missing across uplink |
| Native VLAN | Untagged VLAN on a trunk | Mismatches can create weird reachability/security issues |
| STP | Prevents Layer 2 loops | Loop causes broadcasts and instability |
| LACP | Bundles links into one logical link | Misconfig causes partial or no bundle |
| Port security | Limits allowed MAC behavior | Violation can disable or restrict a port |

## How switches learn

- A switch reads the source MAC address of incoming frames.
- It records which port that MAC came from.
- When the destination MAC is known, the switch forwards only out the correct port.
- When the destination is unknown, the switch floods within the VLAN.

## VLANs and trunks

- A VLAN is a separate Layer 2 broadcast domain.
- Devices in different VLANs need Layer 3 routing to communicate.
- A trunk carries tagged frames for multiple VLANs between switches or between a switch and router/firewall.
- Allowed VLAN lists and native VLAN settings must match the design.

## Loop prevention and redundancy

- Redundant Layer 2 links can create loops if no control protocol blocks a path.
- STP chooses forwarding and blocking roles to keep a loop-free topology.
- EtherChannel/LACP can use multiple physical links as one logical link when configured consistently.

## Scenario walkthrough

1. A phone works on one switch port, but a computer in another office receives an address from the wrong subnet.
2. The likely issue is VLAN assignment on the access port or an incorrect voice/data VLAN setup.
3. Check switchport mode, access VLAN, voice VLAN, and trunk allowance upstream.

## Exam cues

- VLAN mismatch is a common answer when local connectivity fails after a move.
- Trunk problems affect multiple VLANs or one missing VLAN across devices.
- STP is about preventing Layer 2 loops, not routing between subnets.
