Interesting octet: the octet where host and network bits share the same octet (ones and zeros)

Magic number: number of addresses on the network.

| Subnet Mask for Interesting Octet | 128 | 192 | 224 | 240 | 248 | 252 | 254 | 255 |
Calculate the host range, subtract the magic number by two, the network address and the broadcast address.

Identify broadcast address

| IP        | 165  | 245  | 77                      | 14   |
| Subnet ID | 165  | 245  | 64*                     | 0    |
- *this number is calculated by:
  Finding the subnet ranges of that CIDR. take the magic number and increment it from zero to get the bottom number of the ranges, the top number is the the next bottom number minus one.
    0-16, 17-31, 32-47, 48-63, 64-79
    We want the first number of the range that 77 is in, which is 64.

Then we find the network and broadcast addresses using the range we referenced.
Range: 64-79
IP: 165.245.77.14
Interesting octet from subnet mask: the third one
Starting IP (network) = 165.245.64.0
Ending IP (Broadcast) = 165.245.79.255 (notice how we use the last possible address, with the 255)
We have our IP addresses, now we find the host range.
Interesting octet: the third one
Interesting octet of starting IP: 64
For the first IP, 64 You add one and get 65 because 64 is taken by network
First Host IP: 165.245.65.0
For the last IP: we find the last usable IP by taking the broadcast address, that last octet, subtracting the last octet by one
Last Host IP: 165.245.79.254
