## VPC
  Virtual Private Cloud
  Like a VLAN but it's Layer 3
    Handles routing, gateways, and subnets.
    A VLAN that can span across WAN links and VPNs
    VPC Private cloud gateway:
      Connects your homelab from your dad's house to Amazon Web Services, which can back up your homelab in the cloud.
    VPC NAT Gateway
      A gateway that
        - Changes the packets using NAT to allow the NAT Gateway to share one public IP but as a VLAN trunk
        - Remembers traffic requests to go out to the public internet and then to S3, which costs extra money that way
        - Multiple WANs, a gateway with multiple internet providers
        - VPN hardware, that secret sauce
        - Site-to-site to Amazon Web Services
    VPC Endpoint
      - A NAT gateway that accesses Amazon Web Services without a direct connection to the public internet, think of it like a downtown skywalk between buildings (skywalks are really cool)
      - You don't pay for data transfers, so it's like Amazon Prime but you're sending internet traffic instead of packages
    The difference between the three:
      - software configuration and cloud configuration through Amazon Web Services or Azure. The hardware remains the same.
