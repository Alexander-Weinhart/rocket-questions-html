# 1.4 - Ports, Protocols, Services, and Traffic Types

Textbook source: Chapter 1, *Networking Models, Ports, Protocols, and Services*.

Protocols define the rules for communication. Ports identify the service receiving traffic on a host. Traffic types describe the delivery pattern: one-to-one, one-to-many, one-to-nearest, or one-to-all.

## IP, TCP, UDP, and ICMP

Internet Protocol provides logical addressing and delivery across networks. IP is best effort: it sends packets, but it does not guarantee delivery, order, or retransmission by itself.

TCP is connection-oriented. It sets up a session, numbers data, acknowledges receipt, and retransmits missing data. Use TCP when reliable delivery matters.

UDP is connectionless. It has less overhead and does not build the same reliability machinery. Use UDP when speed, simplicity, or real-time behavior matters more than retransmitting every lost piece.

ICMP supports IP with control and error messages. Ping uses ICMP echo request and echo reply. ICMP can also report unreachable destinations and other network conditions.

## IPsec and GRE

IPsec protects IP traffic. Authentication Header helps authenticate packet information. Encapsulating Security Payload can encrypt and authenticate payloads. Internet Key Exchange negotiates security associations and key material.

GRE is a tunneling protocol. It encapsulates traffic so it can be carried across another IP network. GRE does not provide encryption by itself, so secure designs often pair it with IPsec.

## Service Concepts

DNS translates names into addresses and supports other records beyond simple name-to-address lookups. DHCP automatically assigns network configuration such as IP address, mask, gateway, and DNS settings. NTP keeps time synchronized, which matters for logs, certificates, Kerberos, and event correlation.

SNMP is used to monitor and manage network devices. Managed devices expose information through a management information base. Polling retrieves information, and traps send event notifications. SNMPv3 is the version to associate with authentication and encryption.

Syslog sends log messages to a central collector. Centralized logging matters because device-local logs may rotate, disappear, or become unavailable after a failure.

## Port Functions

Ports let one host provide many services at the same IP address. A client connects to an IP address and destination port. The server uses the port number to hand traffic to the correct service.

Well-known ports are defaults, not magic. A service can be moved to a nonstandard port, but the exam usually expects the standard mapping unless the question says otherwise.

## Traffic Types

Unicast is one sender to one receiver. Most ordinary client/server traffic is unicast.

Multicast is one sender to many subscribed receivers. It is more efficient than sending separate unicast copies when a group needs the same data.

Anycast is one sender to the nearest or best instance of a service using the same destination address. It is common in resilient DNS and large distributed services.

Broadcast is one sender to all devices in the local broadcast domain. Broadcasts are useful for discovery but are usually contained by routers.

## Fast Differentiators

TCP is reliable and connection-oriented. UDP is lightweight and connectionless. GRE tunnels but does not encrypt. IPsec protects IP traffic. SSH replaces Telnet. HTTPS protects web traffic. SFTP is not FTP with a tiny lock taped on; it rides over SSH.

## TCP Versus UDP in Real Questions

TCP is the answer when the question emphasizes reliable delivery, ordered data, retransmission, and session establishment. UDP is the answer when the question emphasizes lower overhead, speed, broadcasts, simple queries, or real-time traffic.

DNS often uses UDP for normal queries because the request and response are small. TCP may appear for larger responses or zone transfers. DHCP uses UDP because clients may not even have a usable IP address yet. Voice and video commonly use UDP because late retransmitted audio is not useful.

## Secure and Insecure Pairs

Telnet and SSH are the classic remote-terminal comparison. Telnet is plaintext. SSH is encrypted.

HTTP and HTTPS are the classic web comparison. HTTP is plaintext. HTTPS uses TLS to protect the session.

LDAP and LDAPS are the directory-service comparison. LDAP by itself is not the secure version; LDAPS protects LDAP with SSL/TLS.

FTP and SFTP are often confused. FTP is the older file transfer protocol. SFTP is a file transfer protocol that uses SSH. It is not the same thing as FTPS.

## Management and Monitoring Protocols

SNMP monitors and manages devices. A manager queries agents, and agents expose values through a management information base. Traps let devices send event notifications without waiting to be polled. SNMPv3 is important because it adds stronger authentication and encryption.

Syslog collects logs. It is not a monitoring database by itself; it is a way to send event messages to a central place. Time synchronization with NTP makes those logs useful because events from different systems can be compared accurately.

## Voice and Collaboration

SIP sets up, modifies, and tears down voice or video sessions. It does not carry all media by itself in the way beginners often assume; it is mainly session signaling. This is why voice systems can involve several protocols, but SIP is the one Network+ wants you to recognize for call setup.

## Database and Remote Access Services

SQL Server uses 1433 by default. RDP uses 3389 for remote graphical access to Windows systems. SMB uses 445 for file and printer sharing and is common in Windows networks.

These ports are frequently used in troubleshooting and security questions. If a service works locally but not remotely, the port may be blocked. If a service is exposed when it should not be, the port mapping helps identify the risk.

## Traffic Type Examples

Unicast is a normal client connecting to a web server. Broadcast is a host trying to discover something on the local subnet. Multicast is a stream sent to subscribed receivers. Anycast is a client reaching the nearest instance of a distributed service.

Routers usually contain broadcasts. Multicast requires multicast-aware network support. Anycast depends on routing choosing the best destination among multiple systems advertising the same address.
