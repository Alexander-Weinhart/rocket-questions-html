# 1.7 - IPv4 Network Addressing

Textbook source: Chapter 3, *Network Addressing, Routing, and Switching*.

IPv4 addressing identifies hosts and networks. A host needs enough information to answer two questions: is the destination local, and if not, where should I send the traffic?

## IPv4 Basics

An IPv4 address is 32 bits written as four decimal octets. Each octet represents 8 bits. The address alone is not enough; the subnet mask tells the host which part identifies the network and which part identifies the host.

If the destination is in the same local network, the host sends directly at Layer 2. If the destination is outside the local network, the host sends the packet to its default gateway.

## Address Classes

Classful addressing is older, but Network+ still tests it.

| Class | First octet range | Original purpose |
|---|---:|---|
| A | 1-126 | Very large networks |
| B | 128-191 | Medium networks |
| C | 192-223 | Smaller networks |
| D | 224-239 | Multicast |
| E | 240-255 | Reserved/experimental |

Modern networks use classless addressing, but the class ranges still appear in questions and explanations.

## Subnet Masks and CIDR

A subnet mask separates the network bits from the host bits. A `/24` prefix means 24 network bits and 8 host bits. CIDR notation is the slash-prefix way to write the mask.

Subnetting divides a larger network into smaller networks. This reduces broadcast scope, organizes networks by purpose or location, and uses address space more efficiently.

Variable length subnet masking lets different subnets use different prefix sizes. For example, a point-to-point link does not need the same number of host addresses as a user VLAN.

## Public and Private Addresses

Public addresses are routable on the internet. Private addresses are reserved for internal networks and are not routed directly on the public internet.

| Private range | Common shorthand |
|---|---|
| 10.0.0.0 through 10.255.255.255 | 10.0.0.0/8 |
| 172.16.0.0 through 172.31.255.255 | 172.16.0.0/12 |
| 192.168.0.0 through 192.168.255.255 | 192.168.0.0/16 |

Private addressing usually works with NAT when internal hosts need internet access.

## APIPA and Loopback

APIPA is automatic private IP addressing. If a DHCP client cannot get an address, it can self-assign an address in the APIPA range. APIPA can allow limited local communication with other APIPA hosts, but it usually indicates DHCP failure or lack of network reachability.

Loopback is the local host address range. Testing loopback checks the local TCP/IP stack without crossing the network. If loopback works, the local stack is alive; if remote access fails, keep looking outward.

## Default Gateway

The default gateway is the router used for destinations outside the local subnet. A missing or wrong default gateway commonly produces a pattern where local communication works but remote networks do not.

## Assigning IP Addresses

Addresses can be assigned manually or dynamically. Static addressing is common for infrastructure devices that need predictable addresses. DHCP is common for clients because it reduces manual work and mistakes.

The textbook also discusses the role of DHCP relays in broader networks. DHCP begins as broadcast traffic, and routers do not normally forward broadcasts. A relay or IP helper forwards DHCP requests to a server on another subnet.

## MAC Addresses and ARP

IPv4 identifies the logical destination, but Ethernet still needs a Layer 2 destination MAC address for local delivery. ARP maps IPv4 addresses to MAC addresses on the local network.

This explains why IP and MAC concepts appear together. Routing gets the packet to the right network; local framing gets it to the next hop.

## Scenario Pattern

If a host has a wrong mask, it may think remote devices are local or local devices are remote. If it has the wrong gateway, it may reach local devices but not other networks. If it has APIPA, suspect DHCP or network access failure. If DNS is wrong, IP connectivity may work while names fail.

## Local or Remote Decision

Every IPv4 host uses its address and subnet mask to decide whether a destination is local. If the destination is local, the host uses ARP to find the destination MAC address and sends the frame directly. If the destination is remote, the host uses ARP to find the default gateway's MAC address and sends the frame to the gateway.

That means the frame's destination MAC address may be the gateway even when the packet's destination IP address is a remote server. This is a classic Layer 2/Layer 3 distinction.

## Subnetting Without Panic

Subnetting is borrowing host bits to create more networks. More network bits means more subnets but fewer hosts per subnet. Fewer network bits means larger subnets but fewer separate networks.

The exam may ask for an appropriate subnet size. Start with how many hosts are needed, then choose a prefix that provides enough usable host addresses. Remember that traditional IPv4 subnets reserve network and broadcast addresses, so usable host count is usually total addresses minus two.

## CIDR and Route Summarization

CIDR is not just shorthand. It enables flexible allocation and route summarization. Instead of being trapped by old class A/B/C boundaries, networks can use prefix lengths that match the design.

Route summarization lets multiple networks be represented by a shorter shared prefix. The Domain 1 version of the idea is conceptual: classless addressing is more flexible and efficient than classful addressing.

## Static Versus Dynamic Assignment

Static addressing gives predictability. Infrastructure devices, servers, printers, and network appliances often need stable addresses. Dynamic addressing through DHCP reduces manual configuration and works well for clients.

Misconfigured static addresses can cause duplicates. DHCP failures can lead to APIPA. Wrong DHCP scope options can hand out the wrong gateway or DNS settings. IPv4 troubleshooting often starts with asking how the address was assigned.

## Address Types in Questions

Private addresses are for internal use. Public addresses are internet-routable. Loopback stays on the local host. APIPA is a self-assigned fallback when DHCP fails. Multicast lives in Class D. Class E is reserved.

Those categories matter because a question may give an address and ask what it implies. An APIPA address implies a DHCP or connectivity issue. A loopback address implies local testing. A private address implies NAT is likely needed for internet access.

## DNS Is Not Addressing, But It Gets Blamed

DNS is not part of IPv4 addressing itself, but users often report name-based failures as network failures. If pinging an IP works but browsing by name fails, the address path may be fine and DNS may be the issue. If neither IP nor name works, look at address, mask, gateway, link, and routing first.
