# 1.6 - Network Topologies, Architectures, and Types

Textbook source: Chapter 2, *Network Topologies, Architectures, and Types*.

A topology describes how network devices are arranged and how traffic can flow. Some topologies describe physical cabling. Others describe logical relationships. Good exam answers usually come from comparing resilience, cost, complexity, and traffic patterns.

## Star and Hub-and-Spoke

A star topology connects endpoints to a central device, usually a switch. It is easy to troubleshoot because each device has its own link. If one endpoint cable fails, the rest of the network can keep working. The central device is the important dependency.

Hub-and-spoke uses the same shape at a larger scale. Spoke sites connect back to a hub site. This centralizes control and can simplify security, but spoke-to-spoke traffic may have to pass through the hub unless the design provides another path.

## Mesh

A mesh topology provides multiple paths. Full mesh connects every node to every other node. Partial mesh connects only some nodes with redundant paths.

Mesh improves fault tolerance because traffic can use another path when one link fails. The cost is complexity, cabling, port count, and management overhead. The textbook highlights mesh as strong for redundancy but expensive compared with simpler designs.

## Hybrid

A hybrid topology combines two or more topology patterns. Most real networks are hybrid because an office LAN, a WAN, a data center, and a wireless network do not all need the same design.

Hybrid design is not a failure to choose; it is often the practical answer when different areas have different needs.

## Point-to-Point

Point-to-point is a direct link between two endpoints. It is predictable and simple. A dedicated WAN circuit between two sites is a classic example. Wireless bridges can also behave like point-to-point links.

## Spine-and-Leaf

Spine-and-leaf is common in data centers. Leaf switches connect to servers and access devices. Spine switches connect the leaves. Every leaf connects to every spine, creating predictable paths and reducing bottlenecks.

The design is especially useful for east-west traffic, which is traffic moving between systems inside the data center. Modern applications often have many backend services talking to each other, so the network needs strong internal paths.

## Three-Tier Hierarchical Model

The three-tier model separates a network into access, distribution, and core layers.

The access layer connects endpoint devices. This is where users, printers, phones, and wireless APs attach.

The distribution layer aggregates access switches and applies policy. Routing, filtering, QoS, and summarization often happen here.

The core layer is the high-speed backbone. Its job is fast, reliable transport between major network sections. The core should be simple and resilient because many parts of the network depend on it.

## Collapsed Core

A collapsed core combines the distribution and core layers. Smaller networks may not need a separate core, so collapsing layers reduces cost and complexity. The tradeoff is that the combined layer carries more responsibility.

## Traffic Flows

North-south traffic moves into or out of an environment. User-to-data-center, client-to-internet, and branch-to-cloud traffic are common examples.

East-west traffic moves laterally within an environment. Server-to-server and workload-to-workload communication inside a data center are common examples.

Understanding traffic flow helps explain why older hierarchical designs and newer data center designs do not always optimize for the same thing.

## Older Bus and Ring

Bus topology uses a shared backbone. A failure or termination issue can affect many devices. Ring topology connects devices in a loop. These are older concepts, but the exam may still expect recognition.

## Choosing Between Designs

| Design | Strength | Tradeoff |
|---|---|---|
| Star | Simple endpoint troubleshooting | Central device dependency |
| Mesh | Redundancy and alternate paths | Cost and complexity |
| Hybrid | Fits real mixed environments | Requires careful documentation |
| Point-to-point | Simple dedicated path | Limited flexibility |
| Spine-and-leaf | Predictable data center traffic | More structured switch fabric |
| Three-tier | Scalable enterprise design | More layers to design/manage |
| Collapsed core | Lower cost for smaller networks | Combined layer has more responsibility |

## Physical Versus Logical Topology

A physical topology is how the network is actually cabled or connected. A logical topology is how traffic behaves. A network can be physically star-shaped but logically segmented into many VLANs. A WAN can have physical links that do not match the logical overlay created by tunnels or SD-WAN.

When an exam question describes cabling, ports, or physical links, think physical topology. When it describes how traffic flows, redundancy, or segmentation behaves, think logical topology.

## Redundancy and Single Points of Failure

Star topologies are simple, but the central switch is a single point of failure unless redundancy is added. Mesh topologies reduce single points of failure by adding alternate paths, but they add cost and complexity.

Collapsed core designs can be perfectly reasonable for smaller networks, but they concentrate responsibility. If the collapsed layer fails, the effect can be broad because it is doing both distribution and core work.

## Traffic Flow and Design Choice

North-south-heavy networks can often work well with hierarchical designs because traffic moves from access toward distribution/core and then outward. East-west-heavy environments, such as data centers with many application tiers, benefit from spine-and-leaf because server-to-server paths stay more predictable.

This is the reason the textbook includes traffic flow in the topology chapter: topology is not just shape. It is shape plus traffic behavior.

## Topology Scenario Cues

If the question says maximum redundancy and multiple paths, think mesh. If it says simple direct connection between two locations, think point-to-point. If it says modern data center with predictable low-latency server-to-server communication, think spine-and-leaf. If it says enterprise campus with access, aggregation, and backbone roles, think three-tier. If it says smaller environment combining distribution and core, think collapsed core.

## Why Older Topologies Still Matter

Bus and ring are not common modern Ethernet designs, but they teach failure patterns. A bus depends on a shared medium and proper termination. A ring depends on loop continuity or ring-protection mechanisms. The exam may use them as contrast answers even if you rarely deploy them today.
