# 1.2 - Networking Appliances, Applications, and Functions

Textbook source: Chapter 4, *Network Implementations*.

This objective is about recognizing what common network components do. The exam may describe a need, such as filtering traffic, balancing web requests, centralizing wireless management, or providing shared storage, and expect you to choose the right appliance, application, or function.

## Routers and Switches

A router connects different IP networks. It reads Layer 3 addressing, checks its routing table, and forwards packets toward their destination. Routers are used between subnets, between sites, and at the edge of networks. A host's default gateway is usually a router interface.

A switch connects devices inside a local network. It learns MAC addresses and forwards Ethernet frames only where they need to go. Switches reduce unnecessary traffic compared with old hub-style networks and are the foundation of modern LAN design.

Some enterprise switches can perform Layer 3 functions. This is why exam questions sometimes mention multilayer switches: they switch at Layer 2 but can route between VLANs.

## Security Appliances

A firewall enforces traffic policy. It can allow, deny, or inspect traffic based on rules. Basic filtering may use source, destination, protocol, and port. More advanced firewalls may understand applications, users, content, and threat indicators.

An intrusion detection system watches for suspicious activity and alerts. An intrusion prevention system can actively block or drop malicious traffic. IDS is visibility; IPS is visibility plus enforcement.

A proxy server sits between clients and destinations. It can cache, filter, log, inspect, or hide client details. Because a proxy handles application requests on behalf of clients, it is usually associated with Layer 7 behavior.

## Performance and Availability Appliances

A load balancer distributes requests across multiple servers. This prevents one backend from becoming overloaded and supports high availability. If a server fails a health check, the load balancer can stop sending traffic to it.

Quality of service is a function, not one single box. QoS classifies and prioritizes traffic so delay-sensitive services, such as voice and video, get better treatment during congestion. QoS can involve marking, queuing, shaping, and policing. It does not create bandwidth; it manages contention.

Time to live is another network function. TTL prevents packets from circulating forever. Each router decrements the TTL. When it reaches zero, the packet is discarded. Traceroute-style tools use this behavior to reveal path hops.

## Storage Appliances

Network-attached storage provides file-level storage across the network. Users and systems access shared files rather than raw disks.

A storage-area network provides block-level storage, commonly for servers. To a server, SAN storage can behave more like directly attached disk even though it is delivered across a storage network.

The simple difference: NAS is file sharing; SAN is block storage.

## Wireless Appliances

A wireless access point bridges wireless clients into the wired network. It handles client association and radio access.

A wireless LAN controller centrally manages multiple access points. Controllers are useful when many APs need consistent configuration, roaming support, and coordinated wireless policy.

## Applications and CDNs

A content delivery network distributes content across many locations. The goal is to serve users from a closer or better-performing location instead of forcing everyone back to one origin. CDNs improve latency, absorb demand, and reduce the load on origin infrastructure.

## VPNs

A virtual private network creates a protected tunnel across an untrusted network. Site-to-site VPNs connect networks to networks, such as a branch office to headquarters. Client-to-site VPNs connect an individual user's device to a private network.

IPsec VPNs are strongly associated with network-layer protection. SSL/TLS VPNs are common for remote access because they can use browser-friendly encrypted transport. The exam may focus less on the brand of VPN and more on the purpose: protected connectivity across a network you do not fully trust.

## Physical and Virtual Forms

The textbook discusses appliances, but modern networks often virtualize them. A firewall, load balancer, router, or wireless controller may be a physical device, a virtual appliance, or a cloud-native service. The function matters more than the box shape.

## Quick Comparison

| Need | Best fit |
|---|---|
| Connect different IP networks | Router or Layer 3 switch |
| Connect LAN endpoints | Switch |
| Enforce traffic rules | Firewall |
| Alert on suspicious traffic | IDS |
| Block suspicious traffic | IPS |
| Spread requests across servers | Load balancer |
| Intermediary for client requests | Proxy |
| File-level shared storage | NAS |
| Block-level storage network | SAN |
| Wireless client access | AP |
| Central AP management | Wireless LAN controller |
| Faster distributed content delivery | CDN |
| Protected tunnel | VPN |

## Choosing the Right Device from a Scenario

If the problem is "users in one subnet need to reach another subnet," think router or Layer 3 switch. If the problem is "many local devices need Ethernet connectivity," think switch. If the problem is "traffic must be allowed or denied by policy," think firewall.

If the problem is "we need to know when attacks happen," think IDS. If the problem is "we need to stop attack traffic inline," think IPS. If the problem is "one server is overloaded but several servers can provide the same service," think load balancer.

If the problem is "clients need controlled access to web resources," think proxy. If the problem is "users need shared files," think NAS. If the problem is "servers need block storage over the network," think SAN.

## Appliance Placement

A router often sits at a boundary between networks. A firewall often sits at a trust boundary, such as between internal and external networks or between security zones. A load balancer usually sits in front of a server pool. A proxy sits between clients and the destination service. A wireless LAN controller does not replace APs; it manages them.

Placement matters because some devices only help if traffic actually passes through them. An IPS that is not inline cannot prevent traffic. A load balancer that clients do not use cannot distribute requests. A firewall rule does nothing if the traffic takes a different path.

## NAS Versus SAN

NAS is accessed as files. Users and applications see shares, folders, and files. SAN is accessed as blocks. Servers see storage that behaves more like a disk. This is one of the cleanest exam distinctions in the chapter.

NAS commonly fits file sharing and general shared storage. SAN commonly fits databases, virtualization platforms, and systems that need high-performance block storage.

## Firewall, Proxy, IDS, and IPS Compared

A firewall is about policy enforcement. A proxy is about intermediary application access. IDS is about detection and alerting. IPS is about prevention. These can overlap in modern products, but the exam often separates the roles.

When a question says "inspect web requests on behalf of clients," proxy is stronger than firewall. When it says "alert only," IDS is stronger than IPS. When it says "block malicious traffic automatically," IPS is stronger than IDS.

## Virtualization Angle

The textbook's device list is not limited to rack hardware. A virtual router, virtual firewall, virtual load balancer, or cloud security group can provide the same broad function. Network+ cares that you can identify the role even if the implementation is physical, virtual, or cloud-based.
