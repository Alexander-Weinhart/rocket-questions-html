# 1.3 - Cloud Concepts and Connectivity Options

Textbook source: Chapter 7, *Cloud Computing Concepts and Options*.

Cloud networking uses familiar network ideas in a virtualized provider environment. You still need networks, routes, access rules, gateways, and security boundaries, but they are usually created as software-defined resources.

## Service Models

Software as a service is a complete application delivered by a provider. You use the application and manage your data and user settings, but the provider manages the application platform, servers, storage, and underlying network.

Platform as a service gives developers a managed platform for running code. The provider manages much of the runtime and infrastructure. The customer focuses more on the application than on server maintenance.

Infrastructure as a service gives the customer virtual infrastructure: compute, storage, and networking. The provider manages the physical data center, but the customer manages operating systems, applications, and much of the virtual network design.

| Model | Customer manages more | Provider manages more |
|---|---|---|
| IaaS | OS, applications, data, virtual network choices | Physical hosts, facilities, core platform |
| PaaS | Application code and data | Runtime, platform, OS, infrastructure |
| SaaS | Data, users, configuration | Application and nearly everything below it |

## Deployment Models

Public cloud uses provider-owned infrastructure shared by many customers through logical isolation. It is elastic and widely available.

Private cloud is dedicated to one organization. It can run on premises or in a hosted environment. The key idea is dedicated control and isolation.

Hybrid cloud combines private and public cloud resources. This is common when an organization keeps some workloads close while using public cloud for scale, recovery, or new services.

## Cloud Characteristics

Multitenancy means multiple customers share provider infrastructure while remaining logically separated. This is central to cloud economics.

Elasticity means resources can expand or shrink as demand changes. Scalability means the environment can grow to handle more load. Elasticity is the springy behavior; scalability is the ability to grow.

Network functions virtualization turns network appliances into software functions. Instead of buying a dedicated hardware firewall, router, or load balancer, an organization can deploy a virtualized network function.

## Virtual Private Cloud

A virtual private cloud is a logically isolated network area inside a cloud provider. It usually includes subnets, route tables, gateways, and security controls.

Think of a VPC as the cloud version of carving out your own network space. It does not mean you own the provider's physical infrastructure; it means your cloud resources have a defined logical boundary.

## Cloud Gateways

An internet gateway connects cloud resources to the internet. Without one, a subnet may remain private even if its instances have IP addresses.

A NAT gateway allows private resources to initiate outbound connections without exposing those resources to inbound internet sessions. This is useful for updates, package downloads, or external API access from private systems.

## Security Groups and Security Lists

Network security groups and network security lists are cloud traffic policy tools. They define which traffic is allowed or denied to resources or subnets. They are not exactly the same in every provider, but the exam-level concept is that cloud networks use software policy objects to control reachability.

## Cloud Connectivity

A VPN can connect an on-premises network, branch, or user to a cloud network through an encrypted tunnel. VPNs are flexible and often quick to deploy.

Direct Connect-style services provide private connectivity into the cloud provider. They are chosen when predictable performance, lower latency, or private transport matters.

## Exam Lens

When a question asks how to connect cloud and on-premises networks securely and quickly, think VPN. When it asks for private, predictable provider connectivity, think direct connection. When it asks how private cloud systems reach out without being reachable from the internet, think NAT gateway. When it asks what creates an isolated cloud network, think VPC.

## Shared Responsibility

Cloud does not mean the provider handles everything. Responsibility shifts depending on the model. With SaaS, the provider handles most of the stack. With PaaS, the customer still owns the application and data. With IaaS, the customer has more responsibility for operating systems, host firewalls, routing choices, and virtual network layout.

This matters because a question may ask who secures a virtual machine's operating system in IaaS. That is normally the customer's responsibility. If the question asks who maintains the physical data center, that is the provider.

## Cloud Networking Mental Model

A VPC is the container for your cloud network design. Subnets divide it into smaller network areas. Route tables decide where traffic goes. Gateways connect the environment to the internet, private links, VPNs, or other networks. Security groups and security lists decide what traffic is permitted.

This is the same logic as traditional networking, but expressed in cloud objects.

## Public, Private, and Hybrid Use Cases

Public cloud fits workloads that benefit from provider scale, rapid provisioning, and broad availability. Private cloud fits workloads needing dedicated control, stricter isolation, or custom governance. Hybrid cloud fits mixed environments where some systems stay private while others use public cloud scale.

The exam may not say "choose public cloud." It may say the organization wants elastic capacity without buying hardware. That points public. It may say the organization needs dedicated infrastructure and strict control. That points private. It may say the organization must integrate an internal data center with provider resources. That points hybrid.

## Security Groups Versus Traditional Firewalls

Security groups and lists feel firewall-like, but they are provider-native policy tools. They are often attached to resources or subnets rather than installed as an appliance. Treat them as software-defined access controls that decide which traffic is allowed.

In a scenario, if a cloud instance is running but unreachable, check whether the subnet route, gateway, and security policy all allow the traffic. Cloud connectivity failures often involve more than one object.

## Connectivity Tradeoffs

VPN is encrypted and flexible, but it rides over an underlying network path. Direct provider connectivity is more predictable and private, but it usually requires more planning and cost. NAT gateway is not a site connection; it is outbound internet access for private resources. Internet gateway is not a security policy by itself; it creates an internet path that still needs rules and routing.
