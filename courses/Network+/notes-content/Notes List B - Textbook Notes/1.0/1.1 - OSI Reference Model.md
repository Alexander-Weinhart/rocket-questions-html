# 1.1 - OSI Reference Model

Textbook source: Chapter 1, *Networking Models, Ports, Protocols, and Services*.

The OSI model is a seven-layer map of network communication. Its real value is not trivia; it gives you a way to locate a problem. A disconnected cable, a bad VLAN, an incorrect gateway, a blocked TCP port, and a failed web application all feel like "the network is down," but they live at different layers.

## The Seven Layers

| Layer | Name | Main question | Examples |
|---:|---|---|---|
| 7 | Application | What network service is the software using? | HTTP, DNS, SMTP, SMB, FTP |
| 6 | Presentation | How is the data formatted, encoded, compressed, or encrypted? | TLS/SSL, character encoding, compression |
| 5 | Session | How is the conversation started, maintained, and ended? | Session setup, checkpoints, reconnects |
| 4 | Transport | How do hosts deliver data to the right process? | TCP, UDP, port numbers, segmentation |
| 3 | Network | How does traffic move between networks? | IP addresses, routers, ICMP |
| 2 | Data link | How does traffic move on the local link? | MAC addresses, Ethernet frames, switches, VLANs |
| 1 | Physical | How do bits move across the medium? | Copper, fiber, radio, connectors, signaling |

Layer 1 is the physical reality of the network. If the cable is broken, the fiber is dirty, the wireless signal is unusable, or the connector is wrong, the higher layers never get a fair chance.

Layer 2 is local delivery. Ethernet switches forward frames using MAC addresses. VLANs and trunking also live in this neighborhood because they change how local networks are separated and carried.

Layer 3 is logical addressing and routing. IP addresses, subnet masks, default gateways, and routers belong here. If a host can talk locally but cannot reach another network, Layer 3 deserves attention.

Layer 4 is transport. TCP provides reliable, connection-oriented delivery. UDP provides lightweight, connectionless delivery. Port numbers identify the service or process receiving the traffic.

Layers 5 through 7 are closer to the application. The textbook groups many recognizable services at the top: DNS, SMTP, SMB, FTP, and web protocols are application-layer concerns even though they rely on lower layers to move.

## OSI and TCP/IP

The TCP/IP model is a more compact model used to describe the real internet protocol suite.

| OSI model | TCP/IP model | What it means |
|---|---|---|
| Application, Presentation, Session | Application | User-facing network services and data handling |
| Transport | Transport | TCP/UDP communication and ports |
| Network | Internet | IP addressing and routing |
| Data link, Physical | Network access | Local network delivery and physical transmission |

The OSI model is better for detailed troubleshooting vocabulary. The TCP/IP model is better for describing the practical protocol stack.

## Encapsulation

Encapsulation is the process of wrapping data with layer-specific information as it moves down the stack. Application data becomes a transport segment, then an IP packet, then a data-link frame, then physical bits.

Decapsulation is the reverse. The receiving system removes each wrapper as the data moves up the stack. This is why packet analysis can show multiple nested headers around the same original data.

## Devices by Layer

Devices are often described by the highest layer they primarily understand. A hub is a Layer 1 device because it repeats signals without understanding frames. A switch is usually Layer 2 because it forwards based on MAC addresses. A router is Layer 3 because it forwards based on IP addresses. Firewalls often work at Layers 3 and 4, while next-generation firewalls and proxies may inspect Layer 7 application behavior. Load balancers can work at Layer 4 or Layer 7 depending on whether they balance by transport connection or application content.

## Exam Use

Translate symptoms into layers. No signal or link light is Layer 1. Wrong VLAN or MAC forwarding is Layer 2. Wrong subnet mask or default gateway is Layer 3. Blocked TCP/UDP ports are Layer 4. A service refusing a valid network connection is often Layer 7.

That is the practical magic: the OSI model turns a vague failure into a smaller place to look.

## How Data Changes Names

The textbook's encapsulation discussion is easier if you track the data unit names. At the upper layers, people often just say "data." At Layer 4, that data is commonly discussed as a segment for TCP or a datagram for UDP. At Layer 3, it becomes a packet. At Layer 2, it becomes a frame. At Layer 1, it becomes bits, light pulses, electrical signals, or radio waves.

Those names are not just academic. If a question mentions a malformed frame, think Layer 2. If it mentions a packet with the wrong destination network, think Layer 3. If it mentions TCP sequencing or UDP behavior, think Layer 4.

## Layer-by-Layer Troubleshooting Walkthrough

Imagine a user cannot reach a website. At Layer 1, you check whether the device has a working physical or wireless connection. At Layer 2, you ask whether the device is on the correct local network or VLAN. At Layer 3, you check IP address, subnet mask, gateway, and routing. At Layer 4, you ask whether the right TCP or UDP port is reachable. At Layer 7, you ask whether DNS, HTTP, HTTPS, authentication, or the application itself is failing.

This is why OSI is useful: it turns "website broken" into a chain of testable questions.

## Common Layer Confusions

A MAC address is not a Layer 3 address. MAC addresses are local Layer 2 identifiers. IP addresses are Layer 3 identifiers that can be routed between networks.

A port number is not an IP address. Ports belong to Layer 4 and identify services or processes. An IP address gets traffic to a host or interface; a port helps get it to the right application on that host.

Encryption can appear in different places. TLS is usually discussed above the transport layer because it protects application sessions. IPsec protects at the network layer. The exam may ask this indirectly by describing what part of the communication is protected.

## Exam Patterns

If a question asks which layer a router operates at, answer Layer 3. If it asks about a switch forwarding frames by MAC address, answer Layer 2. If it asks about TCP acknowledgments, answer Layer 4. If it asks about DNS or HTTP, answer Layer 7.

If the question says "which layer should you investigate first," use the symptom. A dead link light starts low. A service-specific failure starts higher. The best answer usually follows the evidence, not a memorized always-start-at-Layer-1 habit.
