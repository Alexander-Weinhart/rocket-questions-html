# 2.2 - Switching Technologies and Features

## Core Idea

Switching technologies segment traffic, control Layer 2 forwarding behavior, and shape how devices communicate inside a local environment.

## VLAN

- Virtual Local Area Network logically separates broadcast domains.
- Improves segmentation, security, and management flexibility.

### VLAN Database

- The switch must know what VLANs exist and how they are named or numbered.

### SVI

- Switch Virtual Interface gives a switch a logical Layer 3 interface tied to a VLAN.
- Used for management or inter-VLAN routing on capable switches.

## Interface Configuration

### Native VLAN

- The VLAN carried untagged on an 802.1Q trunk.
- Native-VLAN mismatches can cause problems and security issues.

### Voice VLAN

- A dedicated VLAN often used for IP phones.
- Helps separate voice and user traffic.

### 802.1Q Tagging

- Adds VLAN information to Ethernet frames on a trunk.
- Lets one link carry multiple VLANs.

### Link Aggregation

- Combines multiple physical links into one logical bundle.
- Increases capacity and resiliency when configured correctly.

### Speed

- Interface speed settings affect link negotiation and throughput expectations.

### Duplex

- Duplex determines whether the link is half-duplex or full-duplex.
- Mismatches can cause serious performance issues.

## Spanning Tree

- Prevents Layer 2 loops in redundant switched topologies.
- Blocks selected redundant paths while preserving failover capability.

## MTU

- Maximum Transmission Unit is the largest packet payload size that can cross a link without fragmentation at that hop.

### Jumbo Frames

- Larger-than-standard Ethernet payload sizes used in some specialized high-performance environments.
- Require end-to-end support where used.

## Big Picture

Switching is not just “plug in and go.” VLAN design, trunking, STP, interface settings, and MTU all affect stability, performance, and segmentation in the LAN.
