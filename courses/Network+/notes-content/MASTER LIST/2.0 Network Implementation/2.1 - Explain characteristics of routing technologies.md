# 2.1 - Routing Technologies

## Core Idea

Routing technologies determine how packets move between networks, how routers learn paths, how they rank those paths, and how they handle redundancy and translation.

## Static Routing

- Manually configured by an administrator.
- Predictable and low overhead.
- Best for simple, stable, or stub designs.
- Requires manual maintenance.
- Poorly scalable in large changing environments.

## Dynamic Routing

- Learned automatically through routing protocols.
- Better for larger or changing networks.
- Reduces manual route management.

### BGP

- Border Gateway Protocol.
- Used between autonomous systems.
- Strongly policy-driven.
- Common at provider and internet-facing boundaries.

### EIGRP

- Enhanced Interior Gateway Routing Protocol.
- Dynamic routing protocol historically associated with Cisco.
- Often described as advanced distance-vector.

### OSPF

- Open Shortest Path First.
- Link-state protocol.
- Common enterprise IGP.
- Good for scalable internal routing designs.

## Route Selection

### Administrative Distance

- Ranks trust between route sources.
- Lower values are preferred.

### Prefix Length

- More specific routes are preferred over less specific ones.
- This is the longest-prefix-match principle.

### Metric

- Used within a routing source to compare path desirability.
- Metric meaning depends on the protocol.

## Address Translation

### NAT

- Network Address Translation rewrites addresses between address domains.
- Commonly used at internet edges.

### PAT

- Port Address Translation lets many internal sessions share fewer public addresses by using transport-layer port differentiation.

## FHRP

- First Hop Redundancy Protocol provides gateway resilience.
- Helps hosts maintain default-gateway availability even if a router fails.

## VIP

- Virtual IP address used as a shared gateway or service endpoint in redundant designs.

## Subinterfaces

- Logical interfaces created under one physical interface.
- Common in router-on-a-stick and segmented routing designs.

## Big Picture

You need to understand how routes are learned, compared, selected, translated, and protected with redundancy. Routing is not just about forwarding. It is also about path choice, failover, and design control.
