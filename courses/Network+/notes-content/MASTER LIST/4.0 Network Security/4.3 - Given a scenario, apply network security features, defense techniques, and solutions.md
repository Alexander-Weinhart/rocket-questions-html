# 4.3 - Security Features, Defense Techniques, and Solutions

## Core Idea

Defensive network design combines hardening, access control, key handling, filtering, and zone separation to reduce attack surface and contain damage.

## Device Hardening

### Disable Unused Ports and Services

- Reduces attack surface.
- Removes unnecessary entry points and listening functions.

### Change Default Passwords

- Default credentials are widely known and commonly abused.

## NAC

- Network Access Control governs who or what may join the network.

### Port Security

- Limits which devices or MAC addresses may use a switch port.

### 802.1X

- Port-based authentication framework for controlled network admission.

### MAC Filtering

- Allows or blocks devices based on MAC address rules.
- Useful but not sufficient alone as a strong identity control.

## Key Management

- Protects generation, storage, rotation, distribution, and use of cryptographic keys.
- Weak key handling undermines otherwise strong encryption.

## Security Rules

### ACL

- Access Control List filters traffic based on rule logic.

### URL Filtering

- Limits access to web destinations based on URL policy.

### Content Filtering

- Blocks or controls access based on content category or policy criteria.

## Zones

### Trusted vs. Untrusted

- Different security expectations and rule sets apply to different zones.

### Screened Subnet

- A segmented zone used to place publicly exposed services while limiting access to more trusted internal resources.

## Defense Principles

- least privilege
- deny by default where appropriate
- segmentation
- verification before access
- minimal exposed services
- layered controls

## Big Picture

Security solutions are strongest when they work together. Hardening reduces exposure, NAC limits entry, key management protects trust, rules control traffic, and zones contain impact.
