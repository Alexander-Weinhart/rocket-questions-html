# 5.5 - Troubleshooting Tools and Protocols

## Core Idea

Good troubleshooting depends on selecting the right tool for the layer, symptom, and scope of the problem.

## Software Tools

### Protocol Analyzer

- Captures and decodes traffic for deep packet inspection.

### Command Line Tools

#### ping

- Tests reachability and basic latency using ICMP.

#### traceroute / tracert

- Maps the Layer 3 path by observing time-exceeded behavior across hops.

#### nslookup

- Queries DNS for name-resolution troubleshooting.

#### tcpdump

- Command-line packet capture and inspection tool.

#### dig

- Detailed DNS query tool.

#### netstat

- Shows local connections, listeners, and protocol statistics.

#### ip / ifconfig / ipconfig

- Shows or changes interface addressing and related local network settings depending on platform.

#### arp

- Displays or interacts with ARP cache information.

### Nmap

- Network scanning and discovery tool.

### LLDP / CDP

- Neighbor discovery protocols that reveal directly connected device information.

### Speed Tester

- Measures performance characteristics such as throughput.

## Hardware Tools

### Toner

- Helps identify cable runs.

### Cable Tester

- Validates cable continuity and wiring conditions.

### Taps

- Provide traffic visibility for monitoring or analysis.

### Wi-Fi Analyzer

- Helps inspect channel use, signal behavior, and wireless interference.

### Visual Fault Locator

- Helps locate fiber issues visibly.

## Basic Networking Device Commands

### show mac-address-table

- Displays learned MAC forwarding entries.

### show route

- Displays routing information.

### show interface

- Displays interface state, counters, and operational details.

### show config

- Displays running or saved configuration depending on platform/context.

### show arp

- Displays Layer 3 to Layer 2 address mappings.

### show vlan

- Displays VLAN definitions and membership.

### show power

- Displays power-related information, often important for PoE troubleshooting.

## Big Picture

Tools are most useful when matched to the problem domain. Start with the symptom, choose the right layer, then pick the smallest tool that reveals the missing truth.
