# 5.3 - Network Service Issues

## Core Idea

Troubleshooting service issues means understanding what dependency is broken: switching behavior, route choice, address allocation, or host configuration.

## Switching Issues

### STP

- Problems in spanning tree can break or destabilize Layer 2 forwarding.

#### Network Loops

- Cause storms, instability, and duplicate traffic.

#### Root Bridge Selection

- Wrong root placement can create inefficient paths.

#### Port Roles

- Incorrect role expectations can reveal STP design or convergence problems.

#### Port States

- Understanding blocked, listening, learning, or forwarding behavior helps isolate STP issues.

### Incorrect VLAN Assignment

- Host or uplink in the wrong VLAN prevents proper communication.

### ACLs

- Filtering rules may be dropping required traffic.

## Route Selection

### Routing Table

- Missing or incorrect routes break reachability.

### Default Routes

- A bad or missing default route can block unknown-destination traffic.

## Address Pool Exhaustion

- Dynamic addressing service may run out of assignable addresses.

## Incorrect Default Gateway

- Hosts may fail to reach remote networks even if local communication works.

## Incorrect IP Address

- Wrong addressing breaks communication and may create conflicting behavior.

### Duplicate IP Address

- Two devices using one address cause intermittent or unstable failures.

## Incorrect Subnet Mask

- Wrong masks cause mistaken local-versus-remote decisions.

## Big Picture

Service troubleshooting is often dependency troubleshooting. Ask whether the problem is at Layer 2, Layer 3, addressing, policy, or route-selection logic.
