# 1.2 - Networking Appliances, Applications, and Functions

## Core Idea

Modern networks rely on hardware and virtual platforms, service applications, and network functions that shape how traffic is forwarded, filtered, accelerated, stored, and secured.

## Physical and Virtual Appliances

### Router

- Connects different IP networks.
- Makes forwarding decisions using Layer 3 information.
- Uses routing tables to choose paths.
- Often serves as the default gateway.

### Switch

- Connects devices inside the same local network.
- Forwards frames using MAC addresses.
- Builds a MAC address table dynamically.
- Can support VLAN segmentation.

### Firewall

- Enforces traffic-control policy.
- Permits, denies, inspects, or logs traffic based on rules.
- Can be stateful or stateless depending on design.
- Often separates trusted and untrusted networks.

### IDS / IPS

- IDS monitors traffic and raises alerts for suspicious behavior.
- IPS actively blocks or drops traffic judged to be malicious.
- Both help detect attacks and policy violations.

### Load Balancer

- Distributes traffic across multiple servers or services.
- Improves scalability and availability.
- Can balance based on Layer 4 or Layer 7 criteria depending on implementation.

### Proxy

- Acts as an intermediary between clients and other services.
- Can filter requests, cache content, hide internal addresses, or enforce policy.
- May be forward-facing for clients or reverse-facing for published services.

### NAS

- Network-attached storage provides file-based shared storage across the network.
- Usually accessed over normal network protocols.
- Common for centralized file shares and backups.

### SAN

- Storage area network provides high-speed storage connectivity.
- Commonly block-based rather than simple file-sharing focused.
- Often used in data center or enterprise storage environments.

### Wireless

#### Access Point (AP)

- Provides wireless access to clients.
- Bridges wireless devices into the wired network.
- Manages SSIDs, radio settings, and client association.

#### Controller

- Centralizes management of multiple APs.
- Helps coordinate policies, firmware, radio tuning, and roaming behavior.

## Applications

### Content Delivery Network (CDN)

- A distributed platform that places content closer to users.
- Reduces latency and server load.
- Improves performance for web content, media, and downloads.

## Functions

### VPN

- Creates protected communication across untrusted networks.
- Used for remote access or site-to-site connectivity.
- Often provides confidentiality and integrity for data in transit.

### QoS

- Prioritizes some traffic over other traffic.
- Helps preserve performance for delay-sensitive applications.
- Commonly used for voice, video, or mission-critical traffic.

### TTL

- Time to live limits how long a packet can remain in the network.
- Prevents packets from looping forever.
- Decrements at Layer 3 hops.

## Physical vs. Virtual

- Physical appliances are dedicated hardware devices.
- Virtual appliances run as software instances on shared compute platforms.
- Virtualization increases flexibility, automation, and portability.
- Physical appliances may still be preferred for performance, dedicated throughput, or specialized hardware.

## How These Fit Together

- Switches handle local forwarding.
- Routers move traffic between networks.
- Firewalls enforce policy.
- IDS/IPS detect and block threats.
- Load balancers spread demand.
- Proxies mediate traffic.
- NAS and SAN handle storage access.
- APs and controllers provide wireless access.
- CDNs accelerate content delivery.
- VPN, QoS, and TTL shape how traffic behaves in transit.

## Big Picture

A strong network engineer should understand not just what each device is called, but what problem it solves, what layer it operates closest to, and how it changes traffic behavior inside a real network.
