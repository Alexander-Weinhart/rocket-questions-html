# 1.4 - Ports, Protocols, Services, and Traffic Types

## Core Idea

Ports, protocols, and services identify how systems communicate. Traffic types describe how packets are addressed and delivered.

## Common Ports and Services

### File and Remote Access

- FTP: `20/21`
- SFTP: `22`
- SSH: `22`
- Telnet: `23`
- RDP: `3389`

### Email and Messaging

- SMTP: `25`
- SMTPS / mail submission: `587`

### Naming and Addressing

- DNS: `53`
- DHCP: `67/68`

### File and Legacy Transfer

- TFTP: `69`
- SMB: `445`

### Web and Encryption

- HTTP: `80`
- HTTPS: `443`

### Time and Monitoring

- NTP: `123`
- SNMP: `161/162`
- Syslog: `514`

### Directory and Database

- LDAP: `389`
- LDAPS: `636`
- SQL Server: `1433`

### Voice and Signaling

- SIP: `5060/5061`

## Why Ports Matter

- Port numbers identify application endpoints at Layer 4.
- They let one host run many services at once.
- Firewalls and ACLs often filter based on ports.
- Troubleshooting often begins with verifying that the expected protocol uses the expected port.

## Internet Protocol Types

### ICMP

- Used for messaging and diagnostics.
- Commonly seen in ping and error signaling.

### TCP

- Connection-oriented.
- Reliable and ordered.
- Uses acknowledgments, retransmissions, sequencing, and flow control.

### UDP

- Connectionless.
- Lower overhead.
- Used where speed or simplicity matters more than built-in reliability.

### GRE

- A tunneling protocol used to encapsulate traffic inside another IP transport path.

### IPSec

- Provides security for IP traffic.
- Commonly used for integrity, authentication, and encryption.

#### AH

- Authentication Header provides integrity and authentication functions.

#### ESP

- Encapsulating Security Payload provides confidentiality and can also provide integrity and authentication.

#### IKE

- Internet Key Exchange helps negotiate security associations and keys used by IPSec.

## Traffic Types

### Unicast

- One sender to one receiver.
- Most normal client-server traffic is unicast.

### Multicast

- One sender to many subscribed receivers.
- Useful when the same data should reach multiple interested hosts efficiently.

### Anycast

- One address used by multiple possible destinations, with delivery typically going to the nearest or best path.
- Common in globally distributed services.

### Broadcast

- One sender to all hosts in the local broadcast domain.
- Broadcast traffic is not forwarded across routers in the normal Layer 2 sense.

## Big Picture

You need to know not only the names of major protocols, but also what they do, what transport they rely on, what port they use when applicable, and how their traffic behaves on the network.
