# 1.8 - Modern Network Environments

## Core Idea

Modern networks are increasingly software-driven, automated, overlay-based, identity-aware, and built to integrate cloud, virtualization, and large-scale distributed environments.

## SDN and SD-WAN

### SDN

- Separates control logic from simple forwarding behavior in many designs.
- Emphasizes centralized control and programmability.

### SD-WAN

- Applies software-driven WAN control across multiple transports.
- Common design goals:
- better path selection
- policy consistency
- easier branch deployment

#### Application Aware

- Traffic can be treated differently based on application identity or quality needs.

#### Zero-touch Provisioning

- New devices can be deployed with minimal manual staging.

#### Transport Agnostic

- Can use MPLS, broadband, LTE, and other transports under the same policy framework.

#### Central Policy Management

- Policies are managed centrally instead of configured independently everywhere.

## VXLAN

- Virtual Extensible LAN is an overlay that extends Layer 2 segments across Layer 3 infrastructure.

### DCI

- Data center interconnect is a major use case for connecting data center segments across routed networks.

### Layer 2 Encapsulation

- VXLAN encapsulates Layer 2 traffic so it can cross Layer 3 boundaries.

## Zero Trust Architecture

- Never assumes trust purely from network location.
- Access decisions should be based on identity, context, and policy.

### Policy-based Authentication

- Access depends on defined trust and verification policies.

### Authorization

- Being authenticated is not enough; permissions must still be granted appropriately.

### Least Privilege Access

- Users and systems should receive only the access they truly need.

## SASE / SSE

- Secure Access Service Edge combines networking and security capabilities in a cloud-delivered model.
- Security Service Edge focuses more tightly on the security-service portion of that model.
- Both aim to move policy closer to users, devices, and applications.

## Infrastructure as Code

- Uses code and templates to manage infrastructure consistently.

### Automation

- Reduces repetitive manual work.

#### Playbooks / Templates / Reusable Tasks

- Standardize deployment and operational actions.

#### Configuration Drift / Compliance

- Helps detect when deployed systems no longer match the intended baseline.

#### Upgrades

- Makes change execution more repeatable.

#### Dynamic Inventories

- Keeps automation aware of changing infrastructure membership.

### Source Control

#### Version Control

- Tracks change history.

#### Central Repository

- Provides a shared source of truth.

#### Conflict Identification

- Helps detect overlapping edits.

#### Branching

- Supports parallel work and controlled integration.

## IPv6 Addressing

### Mitigating Address Exhaustion

- One major driver of IPv6 adoption is the limited size of IPv4 space.

### Compatibility Requirements

#### Tunneling

- Carries IPv6 through IPv4 infrastructure where needed.

#### Dual Stack

- Runs IPv4 and IPv6 together.

#### NAT64

- Supports communication between IPv6-only and IPv4 resources through translation techniques.

## Big Picture

Modern network environments emphasize policy, automation, overlays, identity, cloud integration, and IPv6 readiness. You should understand not just what these technologies are, but why organizations adopt them.
