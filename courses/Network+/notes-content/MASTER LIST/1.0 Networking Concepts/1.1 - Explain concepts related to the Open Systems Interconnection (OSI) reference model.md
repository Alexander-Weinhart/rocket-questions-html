# 1.1 - OSI Reference Model

## Core Idea

The OSI model is a conceptual way to organize networking into layers. Each layer has a focused job, interacts with the layer above and below it, and helps explain how data moves from an application on one host to an application on another host.

## Why the Model Matters

- It gives technicians a shared troubleshooting language.
- It separates responsibilities by function.
- It helps you understand where devices and protocols operate.
- It makes it easier to isolate a failure to a smaller part of the stack.

## Layer 1 - Physical

- Sends raw bits across a medium.
- Defines electrical, optical, radio, and connector behavior.
- Concerns include signaling, cable type, pinout, attenuation, and link speed at the physical medium.
- Typical examples:
- copper cabling
- fiber cabling
- wireless radio signaling
- connectors such as `RJ45`, `LC`, and `SC`

## Layer 2 - Data Link

- Handles local delivery on the same network segment.
- Packages bits into frames.
- Uses hardware addressing such as MAC addresses.
- Detects local frame errors with mechanisms such as FCS.
- Controls how devices share access to a medium.
- Typical devices and concepts:
- switches
- bridges
- NIC MAC addresses
- VLANs
- Ethernet framing

## Layer 3 - Network

- Handles logical addressing and routing between networks.
- Uses IP addresses to identify sources and destinations.
- Decides how packets move between subnets.
- Typical devices and concepts:
- routers
- Layer 3 switches
- IPv4
- IPv6
- routing tables

## Layer 4 - Transport

- Handles end-to-end delivery between applications.
- Uses port numbers.
- Can provide reliability, segmentation, flow control, and session data delivery.
- Main protocol families:
- TCP
- UDP

## Layer 5 - Session

- Establishes, maintains, and terminates communication sessions between applications.
- Coordinates dialog between systems.
- Tracks whether an exchange is started, active, resumed, or closed.

## Layer 6 - Presentation

- Handles data translation so both sides interpret data correctly.
- Deals with format conversion, character encoding, compression, and encryption concepts.
- Makes sure application data is represented in a usable form.

## Layer 7 - Application

- Provides network services directly to user-facing software.
- This is where protocols used by applications live.
- Examples:
- HTTP
- HTTPS
- DNS
- SMTP
- DHCP

## How Data Moves Through the Model

- Data is created at the application side.
- It moves down the stack and is encapsulated.
- Each lower layer adds its own header, and sometimes a trailer.
- At the destination, the process is reversed through decapsulation.

## Encapsulation by Layer

- Layer 7-5: application data
- Layer 4: segment or datagram
- Layer 3: packet
- Layer 2: frame
- Layer 1: bits on the medium

## Troubleshooting with the OSI Model

- Physical issue:
- bad cable
- no link light
- damaged connector
- Data-link issue:
- VLAN mismatch
- MAC-learning issue
- trunking problem
- Network issue:
- wrong IP
- wrong subnet mask
- missing route
- Transport issue:
- blocked port
- TCP session failure
- Application issue:
- service misconfiguration
- DNS problem
- authentication error

## Big Picture

The OSI model does not mean that every protocol maps perfectly into one layer in real life. Its main value is teaching, organizing concepts, and giving you a structured way to think through networking behavior and failures.
