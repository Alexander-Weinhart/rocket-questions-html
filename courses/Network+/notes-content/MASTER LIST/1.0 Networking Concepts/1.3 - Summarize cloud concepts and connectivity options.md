# 1.3 - Cloud Concepts and Connectivity

## Core Idea

Cloud networking extends traditional networking into provider-hosted environments. The key ideas are virtualization, logical segmentation, secure access, and flexible deployment models.

## Network Functions Virtualization (NFV)

- Replaces dedicated hardware appliances with software-based network functions.
- Common functions that can be virtualized:
- firewalls
- routers
- load balancers
- WAN edge functions
- Benefits:
- agility
- easier scaling
- faster deployment
- reduced hardware dependence

## Virtual Private Cloud (VPC)

- A logically isolated cloud network space.
- Lets an organization define private addressing, subnets, routing, and security boundaries inside a public-cloud platform.
- Similar in concept to building a private network inside a provider environment.

## Network Security Groups

- Rule-based security constructs that control permitted traffic.
- Often applied to interfaces, instances, or NICs.
- Commonly used to allow or deny specific protocol, port, source, and destination combinations.

## Network Security Lists

- Another policy mechanism for cloud traffic control.
- Often subnet-oriented instead of instance-oriented, depending on provider.
- Used to regulate inbound and outbound traffic at a broader logical boundary.

## Cloud Gateways

### Internet Gateway

- Connects cloud resources to the public internet.
- Allows internet-routable traffic when the rest of the design also permits it.

### NAT Gateway

- Allows private cloud resources to initiate outbound internet access without directly exposing them to inbound internet sessions.
- Commonly used for updates, repositories, and outbound service access from private subnets.

## Cloud Connectivity Options

### VPN

- Extends encrypted connectivity from an on-premises site to the cloud.
- Often used when dedicated connectivity is unavailable or unnecessary.

### Direct Connect

- A dedicated private connection into a cloud provider.
- Typically offers lower latency, more predictable performance, and stronger private-connectivity characteristics than internet-based VPN alone.

## Deployment Models

### Public Cloud

- Cloud resources owned and operated by a third-party provider.
- Shared provider infrastructure with logical isolation between customers.

### Private Cloud

- Cloud-like environment dedicated to one organization.
- May be hosted on-premises or in dedicated hosted infrastructure.

### Hybrid Cloud

- Combination of on-premises and cloud resources.
- Requires careful integration of identity, security, routing, and application design.

## Design Priorities in Cloud Networking

- segmentation
- secure access
- route control
- identity-aware policy
- connectivity back to on-premises resources
- visibility and logging

## Big Picture

Cloud networking is still networking. The difference is that many controls that used to be physical are now software-defined, policy-driven, and provider-managed through virtual constructs.
