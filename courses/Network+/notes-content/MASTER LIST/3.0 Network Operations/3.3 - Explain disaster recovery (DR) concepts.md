# 3.3 - Disaster Recovery Concepts

## Core Idea

Disaster recovery focuses on how an organization restores service after major disruption. It is about planning for recovery targets, recovery locations, continuity modes, and validation.

## DR Metrics

### RPO

- Recovery Point Objective defines how much data loss is acceptable in time terms.

### RTO

- Recovery Time Objective defines how quickly service must be restored.

### MTTR

- Mean Time To Repair measures average time to restore a failed component or service.

### MTBF

- Mean Time Between Failures measures expected reliability over time.

## DR Sites

### Cold Site

- Minimal readiness.
- Lowest cost.
- Slowest recovery.

### Warm Site

- Partial readiness.
- Moderate cost and moderate recovery speed.

### Hot Site

- High readiness with near-ready infrastructure.
- Highest cost and fastest recovery.

## High-Availability Approaches

### Active-Active

- Multiple systems operate simultaneously.
- Can improve resilience and load sharing.

### Active-Passive

- Primary system carries production load while standby waits to take over.

## Testing

### Tabletop Exercises

- Walk through a disaster scenario and planned response.

### Validation Tests

- Actually test recovery capability in a more practical way.

## Big Picture

Disaster recovery is only useful if objectives are defined clearly and recovery plans are tested. Unverified recovery plans are assumptions, not readiness.
