# 3.4 - IPv4 and IPv6 Network Services

## Core Idea

Network services such as addressing, naming, and time synchronization support nearly every other function in the environment. Misconfiguration here can break whole networks.

## Dynamic Addressing

### DHCP

- Dynamically assigns IPv4 network settings.

#### Reservations

- Keep a device on a consistent assigned address.

#### Scope

- Defines the address pool and associated settings.

#### Lease Time

- Controls how long an address assignment lasts.

#### Options

- Additional parameters such as gateway or DNS information.

#### Relay / IP Helper

- Forwards DHCP requests across routed boundaries.

#### Exclusions

- Addresses intentionally removed from assignment pools.

### SLAAC

- Stateless Address Autoconfiguration lets IPv6 clients self-configure in supported environments.

## Name Resolution

### DNS

- Translates names to addresses and supports many other record-based functions.

#### DNSSEC

- Adds authenticity protections to DNS data.

#### DoH and DoT

- Encrypt DNS queries over HTTPS or TLS.

#### Record Types

##### A

- IPv4 address record.

##### AAAA

- IPv6 address record.

##### CNAME

- Alias record.

##### MX

- Mail server record.

##### TXT

- Text-based metadata record.

##### NS

- Name-server record.

##### PTR

- Pointer record often used for reverse lookup.

#### Zone Types

##### Forward

- Name-to-address lookups.

##### Reverse

- Address-to-name lookups.

#### Authoritative vs. Non-authoritative

- Authoritative answers come from a source responsible for the zone.
- Non-authoritative answers come from a cache or recursive server.

#### Primary vs. Secondary

- Primary holds the writable original zone data.
- Secondary holds replicated copies.

#### Recursive

- Server performs further lookup work on behalf of the client.

### Hosts File

- Local manual name-to-address mapping file.

## Time Protocols

### NTP

- Network Time Protocol synchronizes clocks across systems.

### PTP

- Precision Time Protocol provides finer-grained synchronization in environments that need it.

### NTS

- Network Time Security protects NTP exchanges with added security features.

## Big Picture

If addressing, naming, or time are wrong, authentication, connectivity, logging, and application behavior can all fail. These services are foundational infrastructure.
