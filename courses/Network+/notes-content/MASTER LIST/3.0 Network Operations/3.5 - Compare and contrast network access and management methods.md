# 3.5 - Network Access and Management Methods

## Core Idea

Access and management methods determine how users, administrators, and remote locations reach resources and administer infrastructure securely.

## Site-to-Site VPN

- Connects one network to another across an untrusted path.
- Common for branch, partner, and hybrid-cloud connectivity.

## Client-to-Site VPN

- Lets an individual user connect remotely into an organization.

### Clientless

- Browser-based or portal-based access without a full VPN client in some designs.

### Split Tunnel vs. Full Tunnel

- Split tunnel sends only some traffic through the VPN.
- Full tunnel sends all traffic through the VPN.
- Tradeoff centers on security, bandwidth, privacy, and control.

## Connection Methods

### SSH

- Secure text-based remote management.

### GUI

- Graphical management interface.

### API

- Programmatic management and integration method.

### Console

- Direct local management method often used for initial setup or recovery.

## Jump Box / Host

- Controlled intermediary system used to access sensitive systems.
- Improves management security and auditing.

## In-band vs. Out-of-band Management

- In-band uses the production network path.
- Out-of-band uses a separate management path.
- Out-of-band is valuable during major failures because the production network may be unavailable.

## Big Picture

The best access method depends on security, recovery needs, automation goals, and whether management must remain available when the primary network is impaired.
