# 3.2 - Network Monitoring Technologies

## Core Idea

Monitoring provides visibility into network health, usage, changes, anomalies, and failures before they become larger service problems.

## Monitoring Methods

### SNMP

- Simple Network Management Protocol supports monitoring and management data exchange.

#### Traps

- Unsolicited notifications sent when events occur.

#### MIB

- Management Information Base defines what monitored objects exist and how they are organized.

#### Versions

- Different SNMP versions offer different security and capability models.

#### v2c

- Common older version with community-string-based access.

#### v3

- Adds stronger security features such as authentication and privacy.

#### Community Strings

- Shared identifiers used for access in older SNMP versions.

#### Authentication

- Verifies legitimacy of monitoring access, especially important in v3.

### Flow Data

- Summarizes who is talking to whom and how much traffic is moving.
- Good for traffic behavior analysis without capturing full payloads.

### Packet Capture

- Examines actual packets.
- Useful for protocol-level troubleshooting and forensic analysis.

### Baseline Metrics

- Establish normal expected behavior.

#### Anomaly Alerting / Notification

- Alerts when traffic or performance deviates from the established baseline.

### Log Aggregation

- Centralizes logs from many systems.

#### Syslog Collector

- Receives and stores syslog messages.

#### SIEM

- Security Information and Event Management correlates logs and security events.

### API Integration

- Lets tools exchange monitoring and management information programmatically.

### Port Mirroring

- Copies traffic from one port or VLAN to another for monitoring.

## Monitoring Solutions

### Network Discovery

#### Ad Hoc

- Manual or one-off discovery.

#### Scheduled

- Repeated discovery at planned intervals.

### Traffic Analysis

- Examines who uses bandwidth and how applications behave.

### Performance Monitoring

- Tracks latency, throughput, errors, and device health.

### Availability Monitoring

- Tracks whether systems and services are reachable and operational.

### Configuration Monitoring

- Detects configuration changes and drift.

## Big Picture

Effective monitoring combines telemetry, alerts, baselines, packet visibility, and configuration awareness so teams can move from reactive support to proactive operations.
