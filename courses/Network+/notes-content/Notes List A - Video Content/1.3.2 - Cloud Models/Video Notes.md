## Core Ideas

- When you're deploying a cloud-based application, you also need to consider who will be accessing that application and from where.
- If you're creating an application that anyone on the internet would need to access, then you would probably want to include it in a public cloud so that everyone can gain access.
- But if you're building out a cloud-based application that is for internal use only, you may want to deploy this to your own virtualized local data center, where you have a private cloud.
- In many organizations, we often use both of these.
