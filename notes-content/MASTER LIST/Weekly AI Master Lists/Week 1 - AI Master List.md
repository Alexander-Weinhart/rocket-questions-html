## Canonical Concepts (deduplicated)
1. WAN link: connects LANs; anything outside the local network is treated as WAN.
2. Dedicated WAN connectivity: provider/carrier circuits are used for short-distance dedicated links.
3. Client-to-site VPN: remote client access into internal network resources.
4. Bus topology: one shared backbone cable; all devices share the same medium.
5. Star topology: endpoints connect to a central device.
6. Ring topology: devices form a closed loop.
7. Mesh topology: multiple interconnections between nodes.
8. Hub-and-spoke: centralized style of partial mesh.
9. Partial mesh: some nodes have redundant paths, not all.
10. Full mesh: every node can interconnect; can use internet-based links instead of only dedicated circuits.
11. Interference/noise: signal disturbance that degrades communication.
12. RFI: radio-frequency noise impacting transmissions.
13. EMI: electromagnetic noise affecting signal quality.
14. Fiber degradation: optical medium quality can decline with distance/conditions.
15. Network node: any addressable/participating device in the network.
16. Network core definition: two or more devices connected (wired or wireless) to exchange information.
17. LAN concept: local area network within a building or nearby area; locality defines LAN more than host count.
18. Online/offline state: online means reachable on network; offline means unavailable due to power, link, or config state.
19. Up/down state: up means operational; down means off, failed, or under maintenance.
20. Local vs remote resource: local is on your device; remote resides on another networked system.
21. Network value model: sharing internet access, files, resources, programs, and user communication channels.
22. Shared internet model: one router uplink can provide internet access to multiple internal devices.
23. Shared resource model: printers and storage can be centralized and accessed by multiple users.
24. Server role: provides shared files/services/resources to other devices.
25. Client role: consumes shared resources provided by servers.
26. Dedicated-server model: server is optimized for network services as its primary task.
27. Peer-to-peer model: endpoints can act as both client and server in smaller networks.
28. Core network components: NIC, twisted-pair cable, switch, WAP, router, and firewall.
29. Wireless router composition: common small-network device combining router, firewall, WAP, and switch ports.
30. Security baseline: never expose endpoints directly to the internet without firewall protection.
31. Network-size taxonomy: LAN (local), MAN (metro/city), WAN (large geographic span).
32. Network operations impact: users lose full endpoint independence in shared environments.
33. Network administrator role: manages capacity, backups, access, maintenance, and incident response/escalation.
