## Canonical Concepts (deduplicated)
1. Telnet definition: terminal-emulation protocol for remote CLI administration.
2. Telnet security limitation: cleartext transmission with no encryption.
3. SSH replacement: encrypted remote shell with stronger authentication support.
4. Telnet operational uses: remote management and basic port connectivity testing.
5. Terminal workflow on Windows: enabling Telnet client and running CLI sessions.
6. Cisco switch factory reset path: boot interruption, flash initialization, config deletion, and reboot.
7. Critical reset files: remove startup config and VLAN database; do not remove IOS image files.
8. DNS administration context: name resolution architecture remains core in Week 6 operations.
9. Windows network command usage: command-line tooling for troubleshooting and verification.
10. IPv4 structure: 32-bit address space in four octets.
11. IPv6 structure: 128-bit address space for long-term scalability.
12. IANA role: global authority coordinating IP resource distribution.
13. Allocation chain: IANA to RIR to ISP/organization to endpoints.
14. Classful historical sizing: Class A/B/C host capacity model.
15. Private vs public IP rationale: conserve IPv4 by using private addressing with NAT.
16. RFC1918 private ranges: `10/8`, `172.16/12`, and `192.168/16`.
17. Classless subnetting: CIDR prefix notation decoupled from strict A/B/C boundaries.
18. Prefix-to-mask conversion: `/n` indicates count of network bits set in subnet mask.
19. IP-mask relationship: bitwise masking determines local network membership and gateway decisions.
20. Hexadecimal utility: compact representation used in IP/MAC analysis.
21. BGP function: inter-domain routing and multi-path redundancy between providers.
